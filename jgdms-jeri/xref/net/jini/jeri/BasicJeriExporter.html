43  /**
44   * An <code>Exporter</code> implementation for exporting
45   * a remote object to use Jini extensible remote invocation
46   * (Jini ERI).  Typically, instances of this class should be
47   * obtained from a {@link Configuration} rather than being explicitly
48   * constructed.
49   * Each instance of <code>BasicJeriExporter</code>
50   * can export only a single remote object.
51   *
52   * <p>The following properties (defined during construction) govern
53   * invocation behavior and other characteristics of the exported remote
54   * object and its proxy:
55   * <ul>
56   * <li>{@link ServerEndpoint}: the <code>ServerEndpoint</code> over
57   * which calls are accepted.
58   *
59   * <li><p>{@link InvocationLayerFactory}: a factory used to obtain the remote
60   * object's proxy and invocation dispatcher.
61   *
62   * <li><p><i>enableDGC</i> flag: if <code>true</code>, distributed
63   * garbage collection (DGC) is enabled for the exported remote object,
64   * and the {@link BasicObjectEndpoint} produced by this exporter
65   * participates in DGC and thus constitutes a strong remote reference
66   * to the remote object; if <code>false</code>, DGC is not enabled for
67   * the remote object, and the <code>BasicObjectEndpoint</code> does
68   * not participate in DGC and thus is a weak remote reference, so it
69   * does not prevent the remote object from being garbage collected.
70   *
71   * <li><p><i>keepAlive</i> flag: if <code>true</code>, the virtual
72   * machine is kept alive (with a non-daemon thread) while the remote
73   * object remains exported via this exporter.
74   *
75   * <li><p>{@link Uuid Uuid}: the object identifier to use for the
76   * remote object; if <code>null</code>, a unique object identifier is
77   * chosen for the remote object using {@link UuidFactory#generate
78   * UuidFactory.generate}.
79   * </ul>
80   *
81   * <p>If DGC is not enabled for a remote object, then the
82   * implementation always only weakly references the remote object.  If
83   * DGC is enabled for a remote object, then the implementation weakly
84   * references the remote object when its referenced set is empty and
85   * strongly references the remote object when its referenced set is
86   * not empty (see below).  If the implementation weakly references the
87   * remote object and the weak reference is cleared, the remote object
88   * becomes effectively unexported.
89   * </p>
90   * <p>Enabling DGC is not advisable in some circumstances.  DGC should
91   * not be enabled for a remote object exported with a well known
92   * object identifier.  Enabling DGC with a secure remote object is
93   * generally discouraged, because DGC communication is always made in
94   * a client-side context in which there are no client constraints and
95   * no client subject, so it can leave the remote object open to denial
96   * of service attacks.  Some transport providers may not support making
97   * requests without a client subject, so even if DGC is enabled in the
98   * case where such a transport provider is used, DGC will be effectively
99   * disabled on the client side.
100  * </p>
101  * <p>Multiple remote objects can be exported on the same server endpoint,
102  * and the same remote object can be exported multiple times on different
103  * server endpoints with the only restriction being that a given pair of
104  * object identifier and listen endpoint (derived from the server endpoint)
105  * can only have one active export at any given time.
106  * </p>
107  * <p>Two instances of this class are equal only if they are references to
108  * the same (<code>==</code>) object.
109  * </p>
110  * <p>The server endpoint is not transmitted in the remote reference; only the
111  * derived client endpoint is transmitted.
112  * </p>
113  * <p>Remote objects exported with instances of this class can call {@link
114  * ServerContext#getServerContextElement
115  * ServerContext.getServerContextElement}, passing the class {@link
116  * ClientSubject} to obtain the authenticated identity of the client (if
117  * any) for an incoming remote call, or passing the class {@link
118  * ClientHost} to obtain the address of the client host.
119  * </p>
120  * <p>For remote objects exported with instances of this class, there is no
121  * automatic replacement of the proxy for the remote object during
122  * marshalling; either the proxy must be passed explicitly, or the remote
123  * object implementation class must be serializable and have a
124  * <code>writeReplace</code> method that returns its proxy.
125  * </p>
126  * <h1>Distributed Garbage Collection</h1>
127  * <p>
128  * The <code>BasicJeriExporter</code> class acts as the server-side
129  * DGC implementation for all remote objects exported with DGC enabled
130  * using its instances.
131  * </p>
132  * <p>An entity known as the <i>DGC client</i> tracks the existence
133  * and reachability of live remote references
134  * (<code>BasicObjectEndpoint</code> instances that participate in
135  * DGC) for a <code>BasicObjectEndpoint</code> class in some
136  * (potentially) remote virtual machine.  A DGC client is identified
137  * by a universally unique identifier (a <code>Uuid</code>).  A DGC
138  * client sends <i>dirty calls</i> and <i>clean calls</i> to the
139  * {@link Endpoint} of a live remote reference to inform the
140  * server-side DGC implementation when the number of live remote
141  * references to a given remote object it is tracking increases from
142  * zero to greater than zero and decreases from greater than zero to
143  * zero, respectively.  A DGC client also sends dirty calls to the
144  * <code>Endpoint</code> of live remote references it is tracking to
145  * renew its lease.  The client-side behavior of dirty and clean calls
146  * is specified by {@link BasicObjectEndpoint}.
147  *
148  * <p>On the server side, for every remote object exported with DGC
149  * enabled, the implementation maintains a <i>referenced set</i>,
150  * which contains the <code>Uuid</code>s of the DGC clients that are
151  * known to have live remote references to the remote object.  The
152  * contents of the referenced set are modified as a result of dirty
153  * calls, clean calls, and expiration of leases (see below).  While
154  * the referenced set is not empty, the implementation strongly
155  * references the remote object, so that it will not be garbage
156  * collected.  While the referenced set is empty, the implementation
157  * only weakly references the remote object, so that it may be garbage
158  * collected (if it is not otherwise strongly reachable locally).  If
159  * a remote object is garbage collected, it becomes effectively
160  * unexported.  If a remote object that is an instance of {@link
161  * Unreferenced} is exported with DGC enabled, then whenever the size
162  * of its referenced set transitions from greater than zero to zero,
163  * its {@link Unreferenced#unreferenced unreferenced} method will be
164  * invoked (before the strong reference is dropped).  Note that a
165  * referenced set spans multiple exports of the same (identical)
166  * remote object with <code>BasicJeriExporter</code>.
167  *
168  * <p>For every <code>RequestDispatcher</code> passed by
169  * <code>BasicJeriExporter</code> to a <code>ListenEndpoint</code> as
170  * part of exporting, whenever it has at least one remote object
171  * exported with DGC enabled, it also has an implicitly exported
172  * remote object that represents the server-side DGC implementation.
173  * This remote object is effectively exported with an object
174  * identifier of <code>d32cd1bc-273c-11b2-8841-080020c9e4a1</code> and
175  * an <code>InvocationDispatcher</code> that behaves like a {@link
176  * BasicInvocationDispatcher} with no server constraints, with a
177  * {@link BasicInvocationDispatcher#createMarshalInputStream
178  * createMarshalInputStream} implementation that returns a {@link
179  * MarshalInputStream} that ignores codebase annotations, and with
180  * support for at least the following remote methods:
181  *
182  * <pre>
183  *     long dirty(Uuid clientID, long sequenceNum, Uuid[] ids)
184  *         throws {@link RemoteException};
185  *
186  *     void clean(Uuid clientID, long sequenceNum, Uuid[] ids, boolean strong)
187  *         throws RemoteException;
188  * </pre>
189  *
190  * <code>clientID</code> identifies the DGC client that is making the
191  * dirty or clean call, and <code>sequenceNum</code> identifies the
192  * sequence number of the dirty or clean call with respect to other
193  * dirty and clean calls from the same DGC client.  The sequence
194  * numbers identify the correct order of semantic interpretation of
195  * dirty and clean calls from the same DGC client, regardless of the
196  * order in which they arrive.  The well-known object identifier for
197  * the server-side DGC implementation is reserved; attempting to
198  * export any other remote object with that object identifier always
199  * throws an {@link ExportException}.
200  *
201  * <p>A dirty call is processed as follows:
202  *
203  * <ul>
204  *
205  * <li>It establishes or renews the DGC lease for the identified DGC
206  * client.  The duration of the granted lease, which is chosen by the
207  * implementation, is conveyed as the value returned by the dirty
208  * call, in milliseconds starting from the some time during the
209  * processing of the dirty call.  While the lease for a DGC client is
210  * valid (not expired), the DGC client is preserved in referenced sets
211  * of exported remote objects.
212  *
213  * <li><p>It adds the DGC client's <code>Uuid</code> to the referenced
214  * sets of the exported remote objects identified by <code>ids</code>,
215  * if any, so that they are prevented from being garbage collected.
216  * For each <code>Uuid</code> in <code>ids</code>:
217  *
218  * <blockquote>
219  *
220  * The identified remote object is the remote object exported with
221  * that <code>Uuid</code> on the <code>ListenEndpoint</code> (and thus
222  * <code>RequestDispatcher</code>) that the dirty call was received
223  * on.  If no such exported remote object exists (for example, if it
224  * has been garbage collected), then that <code>Uuid</code> in
225  * <code>ids</code> is ignored.  If the sequence number is less than
226  * the last recorded sequence number of a dirty or clean call for the
227  * identified remote object from the same DGC client, then the remote
228  * object's referenced set is not modified.  Otherwise, the DGC
229  * client's <code>Uuid</code> is added to the remote object's
230  * referenced set (if not already present).  If this addition causes
231  * the referenced set to transition from empty to non-empty, then the
232  * implementation starts strongly referencing the remote object.
233  *
234  * </blockquote>
235  *
236  * </ul>
237  *
238  * <p>A clean call is processed as follows:
239  *
240  * <ul>
241  *
242  * <li>It removes the DGC client's <code>Uuid</code> from the
243  * referenced sets of the exported remote objects identified by
244  * <code>ids</code>, so that they are not prevented from being garbage
245  * collected by the given DGC client.  For each <code>Uuid</code> in
246  * <code>ids</code>:
247  *
248  * <blockquote>
249  *
250  * <p>The identified remote object is the remote object exported with
251  * that <code>Uuid</code> on the <code>ListenEndpoint</code> (and thus
252  * <code>RequestDispatcher</code>) that the dirty call was received
253  * on.  If no such exported remote object exists (for example, if it
254  * has been garbage collected), then that <code>Uuid</code> in
255  * <code>ids</code> is ignored.  If the sequence number is less then
256  * the last recorded sequence number of a dirty or clean call for the
257  * identified remote object from the same DGC client, then the remote
258  * object's referenced set is not modified.  Otherwise, the DGC
259  * client's <code>Uuid</code> is removed from the remote object's
260  * referenced set (if it is present).  If this removal causes the
261  * referenced set to transition from non-empty to empty, then the
262  * implementation starts only weakly referencing the remote object
263  * (and before doing so, if the remote object is an instance of
264  * <code>Unreferenced</code>, its <code>unreferenced</code> method is
265  * invoked).  If <code>strong</code> is <code>true</code>, then a
266  * record is kept of the specified sequence number from the DGC client
267  * for some reasonable period of time, in the event of a dirty call
268  * that might arrive later with a lower sequence number.
269  *
270  * </blockquote>
271  *
272  * </ul>
273  *
274  * <p>If the implementation detects that the most recently granted DGC
275  * lease for a given DGC client has expired, then it assumes that the
276  * DGC client has abnormally terminated, and the DGC client's
277  * <code>Uuid</code> is removed from the referenced sets of all
278  * exported remote objects.  If this removal causes a referenced set
279  * to transition from non-empty to empty, then the implementation
280  * starts only weakly referencing the corresponding remote object (and
281  * before doing so, if the remote object is an instance of
282  * <code>Unreferenced</code>, its <code>unreferenced</code> method is
283  * invoked).
284  *
285  * <p>Unexporting a remote object with a
286  * <code>BasicJeriExporter</code> causes the removal of DGC client
287  * <code>Uuid</code>s from the remote object's referenced set that
288  * were only present because of dirty calls that were received as a
289  * result of exporting it with that <code>BasicJeriExporter</code>.
290  * If the remote object remains exported with DGC enabled by another
291  * <code>BasicJeriExporter</code> and this removal causes the
292  * referenced set to transition from non-empty to empty, then the
293  * implementation starts only weakly referencing the remote object
294  * (and before doing so, if the remote object is an instance of
295  * <code>Unreferenced</code>, its <code>unreferenced</code> method is
296  * invoked).
297  *
298  * <p>When the implementation invokes a remote object's
299  * <code>unreferenced</code> method, it does so with the security
300  * context and context class loader in effect when the remote object
301  * was exported.  If the remote object is currently exported more than
302  * once, then the security context and context class loader in effect
303  * for any one of those exports are used.

485     /**
486      * Exports the specified remote object and returns a proxy for the
487      * remote object.  This method cannot be called more than once to
488      * export a remote object or an <code>IllegalStateException</code> will
489      * be thrown.
490      *
491      * <p>A {@link BasicObjectEndpoint} instance is created with the object
492      * identifier of this exporter, the {@link Endpoint} obtained from
493      * listening on the server endpoint (see below), and the
494      * <code>enableDGC</code> flag of this exporter.
495      *
496      * <p>The client <code>Endpoint</code> for the
497      * <code>BasicObjectEndpoint</code> is obtained by invoking {@link
498      * ServerEndpoint#enumerateListenEndpoints enumerateListenEndpoints} on
499      * the server endpoint with a {@link ServerEndpoint.ListenContext}
500      * whose {@link ServerEndpoint.ListenContext#addListenEndpoint
501      * addListenEndpoint} method is implemented as follows: <ul>
502      *
503      * <li>Invokes {@link ServerEndpoint.ListenEndpoint#checkPermissions
504      * checkPermissions} on the supplied listen endpoint.
505      *
506      * <li>If the supplied listen endpoint has the same class and is equal
507      * to another listen endpoint that has already been listened on,
508      * returns the {@link ServerEndpoint.ListenCookie} corresponding to the
509      * previous <code>listen</code> operation.  Otherwise, it creates a
510      * {@link RequestDispatcher} to handle inbound requests dispatched by
511      * the listen endpoint, invokes {@link
512      * ServerEndpoint.ListenEndpoint#listen listen} on the listen endpoint
513      * (passing the request dispatcher) within an action passed to the
514      * {@link Security#doPrivileged Security.doPrivileged} method, and
515      * returns the <code>ServerEndpoint.ListenCookie</code> obtained by
516      * invoking {@link ServerEndpoint.ListenHandle#getCookie getCookie} on
517      * the {@link ServerEndpoint.ListenHandle} returned from the
518      * <code>listen</code> invocation.
519      * </ul>
520      *
521      * <p>A <code>RequestDispatcher</code> for a listen endpoint handles a
522      * dispatched inbound request (when its {@link
523      * RequestDispatcher#dispatch dispatch} method is invoked) as follows.
524      * The request dispatcher reads the object identifer of the target
525      * object being invoked by invoking {@link UuidFactory#read
526      * UuidFactory.read} on the request input stream of the inbound
527      * request.  If no exported object corresponds to the object identifier
528      * read, it closes the request input stream, writes <code>0x00</code>
529      * to the response output stream, and closes the response output
530      * stream.  Otherwise, it writes <code>0x01</code> to the response
531      * output stream, and invokes the {@link InvocationDispatcher#dispatch
532      * dispatch} method on the invocation dispatcher passing the target
533      * object, the inbound request, and the server context collection (see
534      * below).
535      *
536      * <p>A proxy and an invocation dispatcher are created by
537      * calling the {@link InvocationLayerFactory#createInstances
538      * createInstances} method of this exporter's invocation layer factory,
539      * passing the remote object, the <code>BasicObjectEndpoint</code>, and
540      * the server endpoint (as the {@link ServerCapabilities}). The proxy
541      * is returned by this method. The invocation dispatcher is called for
542      * each incoming remote call to this exporter's object identifier
543      * received from this exporter's server endpoint, passing the remote
544      * object and the {@link InboundRequest} received from the transport
545      * layer.
546      *
547      * <p>Each call to the invocation dispatcher's {@link
548      * InvocationDispatcher#dispatch dispatch} method is invoked with
549      * the following thread context:
550      * <ul>
551      * <li><code>dispatch</code> is invoked in a {@link
552      *     PrivilegedAction} wrapped by a {@link SecurityContext}
553      *     obtained when this method was invoked, with the {@link
554      *     AccessControlContext} of that <code>SecurityContext</code>
555      *     in effect.
556      * <li>The context class loader is the context class loader
557      *     in effect when this method was invoked.
558      * <li>Each call to the dispatcher is made using {@link
559      *     ServerContext#doWithServerContext
560      *     ServerContext.doWithServerContext} with a server context
561      *     collection that is an unmodifiable view of the context
562      *     collection populated by invoking the {@link
563      *     InboundRequest#populateContext populateContext} method on the
564      *     inbound request passing a modifiable collection.  The invocation
565      *     dispatcher's {@link InvocationDispatcher#dispatch dispatch}
566      *     method is invoked with the <code>impl</code>, the inbound
567      *     request, and that modifiable server context collection.
568      * </ul>
569      *
570      * <p>There is no replacement of the proxy for the implementation
571      * object during marshalling; either the proxy must be passed
572      * explicitly in a remote call, or the implementation class must be
573      * serializable and have a <code>writeReplace</code> method that
574      * returns the proxy.
575      *
576      * @throws	ExportException if an object is already exported
577      *		with the same object identifier and server endpoint, or
578      *		the invocation layer factory cannot create a proxy or
579      *		invocation dispatcher, or some other problem occurs while
580      *		exporting the object 
581      * @throws	NullPointerException {@inheritDoc}
582      * @throws	IllegalStateException {@inheritDoc}
583      * @throws	SecurityException if invoking the
584      *		<code>checkPermissions</code> method on any of the listen
585      *		endpoints throws a <code>SecurityException</code>
586      **/
587     public synchronized Remote export(Remote impl)
588 	throws ExportException
589     {
590 	/*
591 	 * Check if object is already exported; disallow exporting more
592 	 * than once via this exporter.
593 	 */
594 	if (used) {
595 	    throw new IllegalStateException(
596 		"object already exported via this exporter");
597 	}
598 	assert (entry == null);		// ()s to work around javadoc bug
599 
600 	/*
601 	 * Export the remote object.
602 	 */
603 	entry = table.export(impl, se, enableDGC, keepAlive, id);
604 	used = true;
605 	
606 	/*
607 	 * Create proxy and invocation dispatcher for the remote object.
608 	 *
609 	 * (Use package-private BasicObjectEndpoint constructor to suppress
610 	 * DGC activity for this local live reference and to keep the impl
611 	 * strongly referenced through it.)
612 	 */
613 	Remote proxy;
614 	InvocationLayerFactory.Instances inst = null;
615 	try {
616 	    ImplContainer implContainer = new ImplContainer(impl);
617 	    weakImplContainer = new WeakReference(implContainer);
618 	    ObjectEndpoint oe =
619 		new BasicObjectEndpoint(entry.getEndpoint(), id,
620 					enableDGC, implContainer);
621 	    inst = ilf.createInstances(impl, oe, se);
622 	    entry.setInvocationDispatcher(inst.getInvocationDispatcher());
623 	    proxy = inst.getProxy();
624 
625 	    if (logger.isLoggable(Level.FINE)) {
626 		logger.log(Level.FINE,
627 			   "export of {0} via {1} returns proxy {2}",
628 			   new Object[]{ impl, this, proxy });
629 	    }
630 	} finally {
631 	    if (inst == null) {
632 		unexport(true);
633 	    }
634 	} 
635 	
636 	return proxy;
637     }

639     /**
640      * Unexports the remote object exported via the exporter's
641      * {@link #export export} method such that incoming remote calls
642      * to the object identifier in this exporter are no longer accepted
643      * through the server endpoint in this exporter.
644      *
645      * <p>If <code>force</code> is <code>true</code>, the object
646      * is forcibly unexported even if there are pending or in-progress remote
647      * calls to the object identifier through the server endpoint.  If
648      * <code>force</code> is <code>false</code>, the object is only
649      * unexported if there are no pending or in-progress remote calls to the
650      * object identifier through the server endpoint.
651      *
652      * <p>The return value is <code>true</code> if the object is (or was
653      * previously) unexported, and <code>false</code> if the object is still
654      * exported.
655      *
656      * @throws	IllegalStateException {@inheritDoc}
657      **/
658     public synchronized boolean unexport(boolean force) {
659 	
660 	if (!used) {
661 	    throw new IllegalStateException(
662 		"no object exported via this exporter");
663 	}
664 
665 	if (entry != null && entry.unexport(force)) {
666 	    entry = null;
667 	    ImplContainer implContainer =
668 		(ImplContainer) weakImplContainer.get();
669 	    if (implContainer != null) {
670 		implContainer.clearImpl();
671 	    }
672 	}
673 
674 	boolean result = entry == null;
675 	if (logger.isLoggable(Level.FINE)) {
676 	    logger.log(Level.FINE, "unexport on {0} returns {1}",
677 		new Object[]{ this, Boolean.valueOf(result) });
678 	}
679 	return result;
680     }

684      * 
685      * @return the string representation for this exporter
686      **/
687     public String toString() {
688 	return "BasicJeriExporter[" + se + "," + id + "]";

691     /**
692      * Container for an impl object.
693      *
694      * BasicJeriExporter, when exporting an impl, passes an impl container
695      * to the package-private BasicObjectEndpoint constructor so that the
696      * BasicObjectEndpoint can reference the impl strongly (through the
697      * container) while the object is exported.  The BasicJeriExporter
698      * instance holds onto the impl container weakly so it won't prevent
699      * the impl from being garbage collected; only the local stub that
700      * references the BasicObjectEndpoint will prevent the impl from being
701      * garbage collected.
702      *
703      * If the object is explicitly unexported via BasicJeriExporter, the
704      * BasicJeriExporter instance clears the impl field (if the container
705      * hasn't been garbage collected) so a reachable stub that references
706      * the container (via the BasicObjectEndpoint in the stub) will not
707      * prevent the impl from being garbage collected.
708      **/
709     private static class ImplContainer {
710 
711 	private Object impl;
712 
713 	ImplContainer(Object impl) {
714 	    this.impl = impl;
715 	}
716 
717 	void clearImpl() {
718 	    impl = null;
719 	}
720     }
