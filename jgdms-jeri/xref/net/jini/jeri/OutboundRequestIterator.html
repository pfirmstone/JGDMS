19  package net.jini.jeri;
20  
21  import java.io.IOException;
22  import java.util.NoSuchElementException;
23  
24  /**
25   * Produces {@link OutboundRequest} instances to use for attempting to
26   * send a particular request to a remote communication endpoint.
27   *
28   * <p>As long as {@link #hasNext hasNext} returns <code>true</code>,
29   * the {@link #next next} method can be invoked to initiate an attempt
30   * to make the request.  If successful, the <code>next</code> method
31   * returns an <code>OutboundRequest</code> to use to write the request
32   * data and read the response.
33   *
34   * <p>If the request communication attempt fails, such as if the
35   * <code>next</code> invocation throws an exception or if a subsequent
36   * I/O operation on the returned <code>OutboundRequest</code> or its
37   * streams throws an exception, then if <code>hasNext</code> returns
38   * <code>true</code> again, the <code>next</code> method can be
39   * invoked again to retry the request attempt.  This process of
40   * retrying failed request attempts can repeat as long as
41   * <code>hasNext</code> returns <code>true</code> after the previous
42   * failed request attempt.
43   *
44   * <p>Note that it is the user's responsibility to abstain from
45   * retrying a request attempt if doing so might violate any applicable
46   * guarantees of <i>at most once</i> execution semantics (invoking
47   * {@link OutboundRequest#getDeliveryStatus getDeliveryStatus} on the
48   * previous <code>OutboundRequest</code> might aid in making that
49   * determination).
50   *
51   * <p>A typical <code>OutboundRequestIterator</code> is likely to
52   * support making only one request attempt, in which case after one
53   * invocation of <code>next</code> (successful or not),
54   * <code>hasNext</code> will return <code>false</code>.  Reasons that
55   * an <code>OutboundRequestIterator</code> might support multiple
56   * request attempts include:
57   *
58   * <ul>
59   *
60   * <li>if the remote endpoint implementation features multiple
61   * communication mechanism alternatives to attempt or
62   *
63   * <li>if the implementation can ascertain that the nature of the
64   * previous request attempt failure indicates that it would be very
65   * unlikely to reoccur in a subsequent attempt.
66   *
67   * </ul>
68   *
69   * The <code>hasNext</code> method should not return <code>true</code>
70   * after successive failed request attempts indefinitely.  The request
71   * retry mechanism provided by <code>OutboundRequestIterator</code> is
72   * not intended for implementing a strategy of general retry of the
73   * same communication mechanism after indeterminate failures.
74   *
75   * <p>Note that it is permitted, although unlikely, for an
76   * <code>OutboundRequestIterator</code>'s <code>hasNext</code> method
77   * to never return <code>true</code>, in which case the
78   * <code>OutboundRequestIterator</code> does not support initiating
79   * even one attempt to send the request.
80   *
81   * <p>An <code>OutboundRequestIterator</code> is not guaranteed to be
82   * safe for concurrent use by multiple threads.
83   *
84   * @author Sun Microsystems, Inc.
85   * @see Endpoint
86   * @since 2.0
87   **/
88  public interface OutboundRequestIterator {
89  
90      /**
91       * Returns <code>true</code> if this iterator supports making at
92       * least one more attempt to communicate the request, and
93       * <code>false</code> otherwise.
94       *
95       * <p>If this method returns <code>true</code>, then it is
96       * guaranteed that the next invocation of {@link #next next} on
97       * this iterator will not throw a {@link NoSuchElementException}.
98       *
99       * <p>If <code>next</code> has been invoked on this iterator and
100      * the previous invocation of <code>next</code> returned an
101      * <code>OutboundRequest</code>, then this method should not be
102      * invoked until that <code>OutboundRequest</code> has been used
103      * to attempt to communicate the request and a failure has
104      * occurred.
105      *
106      * <p>The security context in which this method is invoked may be
107      * used for subsequent verification of security permissions; see
108      * the <code>next</code> method specification for more details.
109      *
110      * @return <code>true</code> if this iterator supports making
111      * another attempt to communicate the request, and
112      * <code>false</code> otherwise
113      **/
114     boolean hasNext();
115 
116     /**
117      * Initiates an attempt to communicate the request to the remote
118      * endpoint.
119      *
120      * <p>After an invocation of {@link #hasNext hasNext} returns
121      * <code>true</code>, it is guaranteed that the next invocation of
122      * this method will not throw a {@link NoSuchElementException}.
123      *
124      * <p>If successful, this method returns an
125      * <code>OutboundRequest</code> to use to write the request data
126      * and read the response.  Even if this method throws an {@link
127      * IOException} or a {@link SecurityException}, the iteration of
128      * attempts to communicate the request may continue with another
129      * invocation of <code>hasNext</code>.
130      *
131      * <p>The implementation verifies that the user's security context
132      * has all of the security permissions necessary to communicate
133      * the current request attempt with the remote endpoint and to
134      * satisfy any required constraints, as appropriate to the
135      * implementation of this interface.  The implementation is
136      * allowed, however, to indicate failure of such a permission
137      * check by either throwing a <code>SecurityException</code> from
138      * this method or, after returning an <code>OutboundRequest</code>
139      * from this method, throwing a <code>SecurityException</code>
140      * from some subsequent operation on the
141      * <code>OutboundRequest</code> or its streams.  If such a
142      * <code>SecurityException</code> is thrown, request data must not
143      * have been transmitted to the server (that is, if an
144      * <code>OutboundRequest</code> has been returned, its {@link
145      * OutboundRequest#getDeliveryStatus getDeliveryStatus} method
146      * must return <code>false</code>), and the client's identity must
147      * not have been revealed to the server.
148      *
149      * <p>Also, the implementation of this method or the returned
150      * <code>OutboundRequest</code> must eventually verify that the
151      * client and server have the requisite principals and credentials
152      * to satisfy any required constraints and if not, throw an
153      * <code>IOException</code>.  If such an <code>IOException</code>
154      * is thrown, request data must not have been transmitted to the
155      * server.
156      *
157      * <p>In verifying any such permission requirement or credential,
158      * the implementation is allowed to use the security context in
159      * effect for this or any previous invocation of a method on this
160      * iterator or the security context in effect for any operation on
161      * the <code>OutboundRequest</code> returned by this or any
162      * previous invocation of this method on this iterator.
163      * Therefore, this iterator and the <code>OutboundRequest</code>
164      * instances that it produces should be used in a uniform security
165      * context.
166      *
167      * @return the <code>OutboundRequest</code> to use to write the
168      * request data and read the response
169      *
170      * @throws NoSuchElementException if this iterator does not
171      * support making another attempt to communicate the request (that
172      * is, if <code>hasNext</code> would return <code>false</code>)
173      *
174      * @throws IOException if an I/O exception occurs while performing
175      * this operation; in this event, the recipient may have received
176      * an indication of the request initiation attempt
177      *
178      * @throws SecurityException if the user does not have the
179      * permissions necessary to communicate with the remote endpoint
180      **/
181     OutboundRequest next() throws IOException;
182 }
