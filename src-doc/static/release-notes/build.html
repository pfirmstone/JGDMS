Release Notes for Apache River Build Mechanism

Build
Apache River v3.0.0 Release Notes

    Description

    This document describes the ASF build mechanism for Apache River.

    Changes since the v2.2.0 release

  • [RIVER-353] - VM sometimes crashes during build when executing jarwrapper
  • [RIVER-92] - preferredlistgen warning messages during build are confusing
  • [RIVER-346] - Logging for QA tests is set to FINEST for many loggers by default . This should be changed to INFO.
  • [RIVER-349] - Setup separate Hudson job for QA testing
  • [RIVER-301] - Move the QA framework donated by Sun inside the jtsk trunk and integrate into build process
  • [RIVER-333] - Tested platforms

    • Pre River 2.2.0 Changes

    Previous to the creation of this document, formal detail of the ASF build mechanism are rather thin. This document provides a starting place for details regarding how the continuious integratio environment at ASF builds the River projects.
