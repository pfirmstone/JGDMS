69  /**
70   * An <code>RMIClassLoader</code> provider that supports preferred
71   * classes.
72   *
73   * <p>See the {@link RMIClassLoader} specification and 
74   * {@link net.jini.loader.ClassLoading} for information
75   * about how to install and configure the <code>RMIClassLoader</code>
76   * service provider.
77   *
78   * <p><code>PreferredClassProvider</code> uses instances of {@link
79   * PreferredClassLoader} to load classes from codebase URI paths
80   * supplied to <code>RMIClassLoader.loadClass</code> methods.  In 
81   * previous releases only codebase URL paths were permitted.
82   *
83   * <p><code>PreferredClassProvider</code> does not enforce {@link
84   * DownloadPermission} by default, but a subclass can configure it to
85   * do so by passing <code>true</code> as the argument to the
86   * <code>protected</code> constructor.
87   *
88   * <p>By overriding the {@link #getClassAnnotation(ClassLoader)
89   * getClassAnnotation(ClassLoader)} method, a subclass can also
90   * configure the class annotations to be used for classes defined by
91   * the system class loader, its ancestor class loaders, and any class
92   * loader that is not an instance of {@link ClassAnnotation} or {@link
93   * URLClassLoader}.
94   *
95   * <h3>Common Terms and Behaviours</h3>
96   *
97   * The following section defines terms and describes behaviours common
98   * to how <code>PreferredClassProvider</code> implements the abstract
99   * methods of <code>RMIClassLoaderSpi</code>.  Where applicable, these
100  * definitions and descriptions are relative to the instance of
101  * <code>PreferredClassProvider</code> on which a method is invoked
102  * and the context in which it is invoked.
103  *
104  * <p>The <i>annotation string</i> for a class loader is determined by
105  * the following procedure:
106  *
107  * <ul>
108  *
109  * <li>If the loader is the system class loader or an ancestor of the
110  * system class loader (including the bootstrap class loader), the
111  * annotation string is the result of invoking {@link
112  * #getClassAnnotation(ClassLoader) getClassAnnotation(ClassLoader)}
113  * with the loader.
114  *
115  * <li>Otherwise, if the loader is an instance of {@link
116  * ClassAnnotation}, the annotation string is the result of invoking
117  * {@link ClassAnnotation#getClassAnnotation getClassAnnotation} on
118  * the loader.
119  *
120  * <li>Otherwise, if the loader is an instance of {@link
121  * URLClassLoader}, the annotation string is a space-separated list of
122  * the URLs returned by an invocation of {@link URLClassLoader#getURLs
123  * getURLs} on the loader.
124  *
125  * <li>Otherwise, the annotation string is the result of invoking
126  * {@link #getClassAnnotation(ClassLoader)
127  * getClassAnnotation(ClassLoader)} with the loader.
128  *
129  * </ul>
130  *
131  * The <i>annotation URL path</i> for a class loader is the path of
132  * URLs obtained by parsing the annotation string for the loader as a
133  * list of URLs separated by spaces, where each URL is parsed as with
134  * the {@link URL#URL(String) URL(String)} constructor; if such
135  * parsing would result in a {@link MalformedURLException}, then the
136  * annotation URL path for the loader is only defined to the extent
137  * that it is not equal to any other path of URLs.
138  *
139  * <p>A <code>PreferredClassProvider</code> maintains an internal
140  * table of class loader instances indexed by keys that comprise a
141  * path of URIs and a parent class loader.  In previous releases keys utilised 
142  * {@link URL}, but now utilise {@link Uri} by default.  The following property 
143  * <code>-Dnet.jini.loader.codebaseAnnotation=URL</code> 
144  * may be set from the command line to revert to {@link URL}.  The table does not
145  * strongly reference the class loader instances, in order to allow
146  * them (and the classes they have defined) to be garbage collected
147  * when they are not otherwise reachable.
148  * 
149  * <p>The behavioural difference between {@link Uri} and {@link URL} when used
150  * in ClassLoader index keys is subtle, {@link URL} remote links rely on DNS to resolve
151  * domain names to IP addresses, for this reason, when using strict {@link URL}
152  * codebase annotations, the IP address of each codebase at the time they're resolved
153  * is part of the codebase annotations identity.  {@link Uri} identity on the other hand is 
154  * determined by RFC3986 normalization and is more flexible, allowing the codebase server 
155  * to change its IP address or be replicated by other codebase servers 
156  * different IP addresses, provided they can be reached by their domain name
157  * address.
158  *
159  * <p>The methods {@link #loadClass loadClass}, {@link #loadProxyClass
160  * loadProxyClass}, and {@link #getClassLoader getClassLoader}, which
161  * each have a <code>String</code> parameter named
162  * <code>codebase</code>, have the following behaviors in common:
163  *
164  * <ul>
165  *
166  * <li><code>codebase</code> may be <code>null</code>.  If it is not
167  * <code>null</code>, it is interpreted as a path of URLs by parsing
168  * it as a list of URLs separated by spaces.  It is recommended that
169  * URLs be compliant with RFC3986 Syntax.  Prior to parsing, any file path 
170  * separators converted to '/' and any illegal characters are percentage escaped,
171  * {@link Uri} is used to parse each URL
172  * in compliance with RFC3986, in addition file URL paths are
173  * converted to upper case for case insensitive file systems. The array of 
174  * RFC3986 normalised Uris along with the current threads context ClassLoader
175  * is used to locate the correct ClassLoader.  After normalisation is complete,
176  * each URL is parsed with the <code>URL(String)</code> constructor; this could 
177  * result in a {@link MalformedURLException}.  This path of URLs is the
178  * <i>codebase URL path</i> for the invocation.
179  *
180  * <li>A class loader known as the <i>codebase loader</i> is chosen
181  * based on <code>codebase</code> and the current thread's context
182  * class loader as follows.  If <code>codebase</code> is
183  * <code>null</code>, then the codebase loader is the current thread's
184  * context class loader.  Otherwise, for each non-<code>null</code>
185  * loader starting with the current thread's context class loader and
186  * continuing with each successive parent class loader, if the
187  * codebase Uri RFC3986 normalised path is equal to the loader's annotation
188  * Uri RFC3986 normalised path, then the codebase loader is that loader.  
189  * If no such matching loader is found, then the codebase loader is the loader 
190  * in this <code>PreferredClassProvider</code>'s internal table with the
191  * codebase Uri RFC3986 normalised path as the key's path of URLs and the current
192  * thread's context class loader as the key's parent class loader.  If
193  * no such entry exists in the table, then one is created by invoking
194  * {@link #createClassLoader createClassLoader} with the codebase URL
195  * path, the current thread's context class loader, and the
196  * <code>boolean</code> <code>requireDlPerm</code> value that this
197  * <code>PreferredClassProvider</code> was constructed with; the
198  * created loader is added to the table, and it is chosen as the
199  * codebase loader.
200  *
201  * <li>The current security context has <i>permission to access the
202  * codebase loader</i> if it has the appropriate permission for each
203  * of the URLs in the codebase loader's annotation URL path, where the
204  * appropriate permission for a URL is defined as follows.  If the
205  * result of invoking {@link URL#openConnection
206  * openConnection()}.{@link URLConnection#getPermission
207  * getPermission()} on the <code>URL</code> object is not a {@link
208  * FilePermission} or if it is a <code>FilePermission</code> whose
209  * name does not contain a directory separator, then that permission
210  * is the appropriate permission.  If it is a
211  * <code>FilePermission</code> whose name contains a directory
212  * separator, then the appropriate permission is a
213  * <code>FilePermission</code> with action <code>"read"</code> and the
214  * same name except with the last path segment replaced with
215  * <code>"-"</code> (that is, permission to read all files in the same
216  * directory and all subdirectories).
217  *
218  * </ul>
219  *
220  * <p>When <code>PreferredClassProvider</code> attempts to load a
221  * class (or interface) named <code><i>N</i></code> using class loader
222  * <code><i>L</i></code>, it does so in a manner equivalent to
223  * evaluating the following expression:
224  *
225  * <pre>
226  * 	Class.forName(<code><i>N</i></code>, false, <code><i>L</i></code>)
227  * </pre>
228  *
229  * In particular, the case of <code><i>N</i></code> being the binary
230  * name of an array class is supported.
231  *
232  * @author Sun Microsystems, Inc.
233  * @since 2.0
234  *
235  * 
236  *
237  * <p>This implementation uses the {@link Logger} named
238  * <code>net.jini.loader.pref.PreferredClassProvider</code> to log
239  * information at the following levels:
240  *
241  * <table summary="Describes what is logged by PreferredClassProvider
242  * at various logging levels" border=1 cellpadding=5>
243  *
244  * <tr> <th> Level <th> Description
245  *
246  * <tr> <td> {@link Levels#FAILED FAILED} <td> class loading failures
247  *
248  * <tr> <td> {@link Levels#HANDLED HANDLED} <td> exceptions caught
249  * during class loading operations
250  *
251  * <tr> <td> {@link Level#FINE FINE} <td> invocations of {@link
252  * #loadClass loadClass} and {@link #loadProxyClass loadProxyClass}
253  *
254  * <tr> <td> {@link Level#FINEST FINEST} <td> detailed activity of
255  * <code>loadClass</code> and <code>loadProxyClass</code>
256  * implementations
257  *
258  * </table>
259  **/

433     /**
434      * Map to hold permissions needed to check the URLs of
435      * URLClassLoader objects.
436      */
437     private final ConcurrentMap<ClassLoader,Collection<Permission>> classLoaderPerms ;

439     /*
440      * Check permissions to load from the specified loader.  The
441      * supplied URLs must be the loader's annotation URLs unless the
442      * loader is identical to the "parent" argument-- this method
443      * checks for permission to access those URLs.
444      *
445      * If the specified loader is identical to the "parent" argument,
446      * no permissions are checked, because the caller could have used
447      * an empty codebase to achieve the same effect anyway.
448      */
449     private void checkLoader(ClassLoader loader, ClassLoader parent, Uri[] uris,
450 			     URL[] urls)
451     {
452 	SecurityManager sm = System.getSecurityManager();
453 
454 	if ((sm != null) && (loader != null) && (loader != parent)) {
455 	    assert urlsMatchLoaderAnnotation(uris, loader);
456 
457 	    if (loader.getClass() == PreferredClassLoader.class) {
458 		((PreferredClassLoader) loader).checkPermissions();
459 		
460 	    } else {
461 		Collection<Permission> perms = classLoaderPerms.get(loader);
462                 if (perms == null) {
463 		    PermissionCollection permiss = new Permissions();
464                     // long operation so we don't want to synchronize here.
465                     PreferredClassLoader.addPermissionsForURLs(
466                         urls, permiss, false);
467 		    perms = new LinkedList<Permission>();
468 		    Enumeration<Permission> en = permiss.elements();
469 		    while(en.hasMoreElements()){
470 			perms.add(en.nextElement());
471 		    }
472                     Collection<Permission> existed = 
473 			    classLoaderPerms.putIfAbsent(loader, perms);
474 		    // It doesn't matter if perms existed, but this keeps 
475 		    // FindBugs happy.
476 		    if (existed != null) perms = existed;
477                 }
478                 Iterator<Permission> it = perms.iterator();
479                 while (it.hasNext()) {
480                     sm.checkPermission(it.next());
481                 }
482 	    }
483 	}

486     /**
487      * Provides the implementation for {@link
488      * RMIClassLoaderSpi#loadClass(String,String,ClassLoader)}.
489      *
490      * <p><code>PreferredClassProvider</code> implements this method
491      * as follows:
492      *
493      * <p>If <code>name</code> is the binary name of an array class
494      * (of one or more dimensions) with a primitive element type, this
495      * method returns the <code>Class</code> for that array class.
496      *
497      * <p>Otherwise, if <code>defaultLoader</code> is not
498      * <code>null</code> and any of the following conditions are true:
499      *
500      * <ul>
501      *
502      * <li>There is no security manager.
503      *
504      * <li>The codebase loader is not the current thread's context
505      * class loader and the current security context does not have
506      * permission to access the codebase loader.
507      *
508      * <li><code>codebase</code> is <code>null</code>.
509      *
510      * <li>The specified codebase URL path is equal to the annotation
511      * URL path of <code>defaultLoader</code>.
512      *
513      * <li>The codebase loader is not an instance of {@link
514      * PreferredClassLoader}.
515      *
516      * <li>The codebase loader is an instance of
517      * <code>PreferredClassLoader</code> and an invocation of {@link
518      * PreferredClassLoader#isPreferredResource isPreferredResource}
519      * on the codebase loader with the class name described below as
520      * the first argument and <code>true</code> as the second argument
521      * returns <code>false</code>.  If <code>name</code> is the binary
522      * name of an array class (of one or more dimensions) with a
523      * element type that is a reference type, the class name passed to
524      * <code>isPreferredResource</code> is the binary name of that
525      * element type; otherwise, the class name passed to
526      * <code>isPreferredResource</code> is <code>name</code>.  This
527      * invocation is only done if none of the previous conditions are
528      * true.  If <code>isPreferredResource</code> throws an
529      * <code>IOException</code>, this method throws a
530      * <code>ClassNotFoundException</code>.
531      *
532      * </ul>
533      *
534      * then this method attempts to load the class with the specified
535      * name using <code>defaultLoader</code>.  If this attempt
536      * succeeds, this method returns the resulting <code>Class</code>;
537      * if it throws a <code>ClassNotFoundException</code>, this method
538      * proceeds as follows.
539      *
540      * <p>Otherwise, this method attempts to load the class with the
541      * specified name using the codebase loader, if there is a
542      * security manager and the current security context has
543      * permission to access the codebase loader, or using the current
544      * thread's context class loader otherwise.  If this attempt
545      * succeeds, this method returns the resulting <code>Class</code>;
546      * if it throws a <code>ClassNotFoundException</code>, this method
547      * throws a <code>ClassNotFoundException</code>.
548      *
549      * @param codebase the codebase URL path as a space-separated list
550      * of URLs, or <code>null</code>
551      *
552      * @param name the binary name of the class to load
553      *
554      * @param defaultLoader additional contextual class loader
555      * to use, or <code>null</code>
556      *
557      * @return the <code>Class</code> object representing the loaded class
558      *
559      * @throws MalformedURLException if <code>codebase</code> is
560      * non-<code>null</code> and contains an invalid URL
561      *
562      * @throws ClassNotFoundException if a definition for the class
563      * could not be loaded
564      **/
565     @Override
566     public Class loadClass(String codebase,
567 			   String name,
568 			   ClassLoader defaultLoader)
569 	throws MalformedURLException, ClassNotFoundException
570     {
571 	if (logger.isLoggable(Level.FINE)) {
572 	    logger.log(Level.FINE, //logging here can cause deadlock
573 		       "name=\"{0}\", codebase={1}, defaultLoader={2}",
574 		       new Object[] {
575 			   name,
576 			   codebase != null ? "\"" + codebase + "\"" : null,
577 			   defaultLoader
578 		       });
579 	}
580         
581         // throws MalformedURLException
582     	Uri[] codebaseURIs = pathToURIs(codebase);	// may be null
583         URL[] codebaseURLs = asURL(codebaseURIs); // throws MalformedURLException
584 
585 	/*
586 	 * Process array class names.
587 	 */
588 	String elementTypeName = null;
589 	int len = name.length();

1578     /**
1579      * Return the origin class loader for the <code>pathURLs</code> or
1580      * null if the origin loader was not present in the delegation
1581      * hierarchy.
1582      *
1583      * Preferred classes introduces the "class boomerang" problem to
1584      * RMI class loading.  A class boomerang occurs when a class which
1585      * is marked preferred is accessible from the codebase of a
1586      * virtual machine (VM) and is loaded by that VM.  Since the VM
1587      * has a copy of the class in its own resources and the class is
1588      * "returning to its origin" the class should not be preferred.
1589      * If the class is preferred, it will be loaded in a class loader
1590      * for the local codebase annotation.  As a result, the class'
1591      * type will not be compatible with types defined from the local
1592      * definition of the class file in the relevant VM.
1593      *
1594      * A boomerang can also occur when a new child loader for a URL
1595      * path is created but an ancestor of the new class loader has the
1596      * same URL path as the new class loader.  In such cases the new
1597      * class loader should not be created.  The incoming class should
1598      * be loaded from the origin ancestor instead.
1599      *
1600      * A simple example of a class boomerang occurs when when a VM
1601      * makes a remote method call to itself.  Suppose an object whose
1602      * class was loaded locally in that VM and is preferred in the
1603      * codebase of the VM is passed in the call.  When the VM receives
1604      * its own call, an instance of the unmarshalled parameter will
1605      * not be assignable to instances that were defined by local
1606      * classes (i.e. never unmarshalled).
1607      *
1608      * In order to work around class boomerang problems, the preferred
1609      * class provider lookupLoader algorithm is different from the
1610      * analogous algorithm in LoaderHandler.  To avoid boomerangs, the
1611      * lookupLoader method of this class attempts to locate the
1612      * "origin" class loader of an incoming class in a remote method
1613      * call.  Loading classes from their origin loader instead of in a
1614      * preferred circumvents type compatibility conflicts.
1615      *
1616      * To find origin loaders, the lookupLoader method calls
1617      * findOriginLoader() before locating or creating new
1618      * PreferredClassLoaders.  An origin loader is found by searching
1619      * up the delegation hierarchy above the parent (context) class
1620      * loader for a loader that has an export path which matches the
1621      * parameter path.
1622      */
1623     private ClassLoader findOriginLoader(final Uri[] pathURLs,
1624 					 final ClassLoader parent)
1625     {
1626 	return AccessController.doPrivileged(
1627             new PrivilegedAction<ClassLoader>() {
1628 	      @Override
1629               public ClassLoader run() {
1630                   return findOriginLoader0(pathURLs, parent);
1631               }
1632             }
1633         );
1634     }
