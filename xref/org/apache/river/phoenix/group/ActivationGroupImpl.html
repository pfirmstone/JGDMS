37  /**
38   * The default activation group implementation for phoenix.  Instances of
39   * this class are configurable through a {@link Configuration}, as detailed
40   * further below, and provide the necessary support to allow exporter-based
41   * remote objects to go inactive.  Instances of this class support the
42   * creation of remote objects through the normal activatable constructor;
43   * an activatable remote object must either implement the {@link
44   * ProxyAccessor} interface to return a suitable proxy for the remote
45   * object, or the remote object must itself be serializable and marshalling
46   * the object must produce a suitable proxy for the remote object.

49   * {@link org.apache.river.phoenix.common.ActivationGroupData} instance containing configuration options

173     /**
174      * Creates an {@link java.rmi.activation.ActivationGroup} instance and
175      * returns it. An {@link org.apache.river.phoenix.common.ActivationGroupData} instance is extracted from
176      * the initialization data, and a {@link Configuration} is obtained by
177      * calling
178      * {@link net.jini.config.ConfigurationProvider#getInstance
179      * Configuration.Provider.getInstance} with the configuration options from
180      * that instance. A {@link LoginContext} is obtained from the
181      * <code>loginContext</code> configuration entry, if one exists; if the
182      * value is not <code>null</code>, a login is performed on that context,
183      * and the resulting {@link Subject} (set to be read-only) is used as the
184      * subject when executing the rest of this method. The subject is also
185      * used for all subsequent remote calls by this class to the
186      * {@link ActivationMonitor}. The {@link ActivationSystem} proxy
187      * (obtained from the <code>ActivationGroupID</code>) is passed to the
188      * {@link ProxyPreparer} given by the <code>systemPreparer</code>
189      * configuration entry, if one exists; a new
190      * <code>ActivationGroupID</code> is constructed with the resulting proxy.
191      * An {@link Exporter} instance is obtained from the
192      * <code>instantiatorExporter</code> configuration entry, if one exists;
193      * this exporter will be used (in the constructor of this class) to export
194      * the group. A <code>ProxyPreparer</code> instance is obtained from the
195      * <code>monitorPreparer</code> configuration entry, if one exists; this
196      * preparer will be used (in the constructor of this class) to prepare the
197      * <code>ActivationMonitor</code>. A call is then made to
198      * {@link ActivationGroup#createGroup ActivationGroup.createGroup} with
199      * the new group identifier, the activation group descriptor, and the
200      * group incarnation number, and the result of that call is returned.
201      *
202      * @param id the activation group identifier
203      * @param desc the activation group descriptor
204      * @param incarnation the group's incarnation number (zero on initial
205      * creation)
206      * @return the created activation group
207      * @throws ActivationException if a group already exists or if an
208      * exception occurs during group creation
209      */
210     public static synchronized
211 	java.rmi.activation.ActivationGroup createGroup(
212 					      final ActivationGroupID id,
213 					      final ActivationGroupDesc desc,
214 					      final long incarnation)
215         throws ActivationException
216     {
217         return AbstractActivationGroup.createGroup(id, desc, incarnation);
218     }
219       
220     /**
221      * Creates an instance with the specified group identifier and
222      * initialization data. This constructor must be called indirectly,
223      * via {@link #createGroup createGroup}. By default, this instance
224      * automatically exports itself as a {@link UnicastRemoteObject}. (This
225      * is a limitation of the existing activation system design.) If an
226      * {@link Exporter} was obtained by {@link #createGroup createGroup},
227      * then this instance is unexported from the JRMP runtime and re-exported
228      * using that exporter. (Any incoming remote calls received on the
229      * original JRMP export before this instance can be unexported will be
230      * refused with a security exception thrown.) The
231      * {@link ActivationSystem#activeGroup activeGroup} method of the
232      * activation system proxy (in the group identifier) is called to
233      * make the group active. The returned {@link ActivationMonitor} proxy
234      * is passed to the corresponding {@link ProxyPreparer} obtained by
235      * <code>createGroup</code>. Note that after this constructor returns,
236      * {@link ActivationGroup#createGroup ActivationGroup.createGroup} will
237      * also call <code>activeGroup</code> (so the activation system must
238      * accept idempotent calls to that method), but the
239      * <code>ActivationMonitor</code> proxy returned by that call will not be
240      * used.
241      * 
242      * @param id the activation group identifier
243      * @param data group initialization data (ignored)
244      * @throws RemoteException if the group could not be exported or
245      * made active, or proxy preparation fails
246      * @throws ActivationException if the constructor was not called
247      * indirectly from <code>createGroup</code>
248      */
249     public ActivationGroupImpl(ActivationGroupID id, MarshalledObject data)
250 	throws ActivationException, RemoteException
251     {
252 	super(id, data);
253         export();
254     }
