29  /**
30   * This class encapsulates the functionality required of an entity that
31   * wishes to employ the unicast discovery protocol to discover a lookup
32   * service. This utility provides an implementation that makes the process
33   * of finding specific lookup services much simpler for both services and
34   * clients.
35   * <p>
36   * Because this class participates in only the unicast discovery protocol,
37   * and because the unicast discovery protocol imposes no restriction on the
38   * physical location of the entity relative to a lookup service, this utility
39   * can be used to discover lookup services running on hosts that are located
40   * far from, or near to, the host on which the entity is running. This lack
41   * of a restriction on location brings with it a requirement that the
42   * discovering entity supply this class with specific information about the
43   * desired lookup services; namely, the location of the device(s) hosting
44   * each lookup service. This information is supplied through an instance
45   * of the {@link net.jini.core.discovery.LookupLocator LookupLocator} class,
46   * or its subclass {@link net.jini.discovery.ConstrainableLookupLocator
47   * ConstrainableLookupLocator}.
48   *

53   *     <li> <a href="#lldLogging">Logging</a>
54   * </ul>

56   * <a name="lldConfigEntries"><b>Configuring LookupLocatorDiscovery</b></a>
57   *
58   * This implementation of <code>LookupLocatorDiscovery</code> supports the
59   * following configuration entries; where each configuration entry name
60   * is associated with the component name
61   * <code>net.jini.discovery.LookupLocatorDiscovery</code>. Note that the
62   * configuration entries specified here are specific to this implementation
63   * of <code>LookupLocatorDiscovery</code>. Unless otherwise stated, each
64   * entry is retrieved from the configuration only once per instance of
65   * this utility, where each such retrieval is performed in the constructor.
66   *

73   *     <code>initialUnicastDelayRange</code>

75   *   <tr valign="top"> <td> &nbsp; <th scope="row" align="right">
76   *     Type: <td> <code>long</code>
77   * 
78   *   <tr valign="top"> <td> &nbsp; <th scope="row" align="right">

80   * 
81   *   <tr valign="top"> <td> &nbsp; <th scope="row" align="right">
82   *     Description:
83   *       <td> With respect to when this utility is started, this entry controls
84   *       how long to wait before attempting unicast discovery.
85   *       If the value is positive, initial unicast discovery requests
86   *       will be delayed by a random value between <code>0</code> and
87   *       <code>initialUnicastDelayRange</code> milliseconds. Once the wait
88   *       period is up, the <code>LookupLocator</code>s specified at construction
89   *       time are used for initiating unicast discovery requests, unless the

91   *       in which case, no delayed unicast discovery requests are performed.
92   *       Note that this entry only has effect when this utility is initialized.
93   *       It does not delay discovery requests that are initiated if the managed
94   *       <code>LookupLocator</code>s are subsequently changed.

257  * <p>
258  *
259  * This implementation of <code>LookupLocatorDiscovery</code> determines
260  * the constraints (if any) to apply to unicast discovery for a given
261  * {@link net.jini.core.discovery.LookupLocator LookupLocator} instance
262  * by calling the 
263  * {@link net.jini.core.constraint.RemoteMethodControl#getConstraints
264  * getConstraints} method of that instance, if it implements the
265  * {@link net.jini.core.constraint.RemoteMethodControl RemoteMethodControl}
266  * interface. If the {@link net.jini.core.discovery.LookupLocator
267  * LookupLocator} instance does not implement
268  * {@link net.jini.core.constraint.RemoteMethodControl RemoteMethodControl},
269  * then no constraints are applied to unicast discovery for that instance.
270  * <p>
271  * For more information on constraining unicast discovery, refer to the
272  * documentation for the {@link net.jini.discovery.ConstrainableLookupLocator
273  * ConstrainableLookupLocator} class.
274  *
275  * @author Sun Microsystems, Inc.
276  *
277  * @see net.jini.core.discovery.LookupLocator
278  */
279 public final class LookupLocatorDiscovery extends AbstractLookupLocatorDiscovery 
280                                     implements DiscoveryManagement,
281                                                DiscoveryLocatorManagement
282 {
283     /**
284      * Creates an instance of this class (<code>LookupLocatorDiscovery</code>),
285      * with an initial array of <code>LookupLocator</code>s to be managed.
286      * For each managed <code>LookupLocator</code>, unicast discovery is
287      * performed to obtain a <code>ServiceRegistrar</code> proxy for that
288      * lookup service.
289      * 
290      * @param locators the locators to discover
291      * 
292      * @throws java.lang.NullPointerException input array contains at least
293      *         one <code>null</code> element
294      */
295     public LookupLocatorDiscovery(LookupLocator[] locators) {
296         super();
297         super.beginDiscovery(locators);
298     }//end constructor
299 
300     /**
301      * Constructs a new lookup locator discovery object, set to discover the
302      * given set of locators, and having the given <code>Configuration</code>.
303      * <p>
304      * For each managed <code>LookupLocator</code>, unicast discovery is
305      * performed to obtain a <code>ServiceRegistrar</code> proxy for that
306      * lookup service.
307      * 
308      * @param locators the locators to discover
309      *
310      * @param config   an instance of <code>Configuration</code>, used to
311      *                 obtain the objects needed to configure the current
312      *                 instance of this class
313      *
314      * @throws net.jini.config.ConfigurationException indicates an exception
315      *         occurred while retrieving an item from the given
316      *         <code>Configuration</code>
317      * 
318      * @throws java.lang.NullPointerException input array contains at least
319      *         one <code>null</code> element or <code>null</code> is input
320      *         for the configuration
321      */
322     public LookupLocatorDiscovery(LookupLocator[] locators,
323                                   Configuration config)
324                                                 throws ConfigurationException
325     {
326         super(config);
327         super.beginDiscovery(locators);
328     }//end constructor
329 
330 }//end class LookupLocatorDiscovery
