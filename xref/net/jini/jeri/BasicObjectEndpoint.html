57  /**
58   * References a remote object with an {@link Endpoint Endpoint} for
59   * sending requests to the object and a {@link Uuid Uuid} to identify
60   * the object at that <code>Endpoint</code>.
61   *
62   * <p>In addition to the <code>Endpoint</code> and the
63   * <code>Uuid</code>, <code>BasicObjectEndpoint</code> instances also
64   * contain a flag indicating whether or not the instance participates
65   * in distributed garbage collection (DGC).
66   * </p>
67   * <p>The {@link #newCall newCall} method can be used to send a
68   * request to the remote object that this object references.
69   * </p>
70   * <h1>Distributed Garbage Collection</h1>
71   * <p>
72   * The <code>BasicObjectEndpoint</code> class acts as the <i>DGC
73   * client</i> for all of its instances that participate in DGC (which
74   * are called live remote references).  That is, it tracks the
75   * existence and reachability of live remote references and makes
76   * <i>dirty calls</i> and <i>clean calls</i> to the associated
77   * server-side DGC implementations, as described below.
78   * </p>
79   * <p>The server-side behavior of dirty and clean calls is specified
80   * by {@link BasicJeriExporter}.  When the DGC client makes a dirty or
81   * clean call to a given <code>Endpoint</code>, the behavior is
82   * effectively that of using a <code>BasicObjectEndpoint</code>
83   * containing that <code>Endpoint</code> and the object identifier
84   * <code>d32cd1bc-273c-11b2-8841-080020c9e4a1</code> (and that doesn't
85   * itself participate in DGC), wrapped in a {@link
86   * net.jini.jeri.BasicInvocationHandler} with no client or server
87   * constraints, wrapped in an instance of a dynamic proxy class that
88   * implements an interface with the following remote methods:
89   *
90   * <pre>
91   *     long dirty(Uuid clientID, long sequenceNum, Uuid[] ids)
92   *         throws {@link RemoteException};
93   *
94   *     void clean(Uuid clientID, long sequenceNum, Uuid[] ids, boolean strong)
95   *         throws RemoteException;
96   * </pre>
97   *
98   * <code>clientID</code> is the DGC client's universally unique
99   * identifier, which is generated using {@link UuidFactory#generate
100  * UuidFactory.generate}.  <code>sequenceNum</code> identifies the
101  * sequence number of the dirty or clean call with respect to all
102  * other dirty and clean calls made by the same DGC client (regardless
103  * of the <code>Endpoint</code> that the calls are made to).  All
104  * dirty and clean calls made by a DGC client must have a unique
105  * sequence number that monotonically increases with the temporal
106  * order of the states (of reachable live remote references) that the
107  * calls assert.  A dirty call asserts that live remote references
108  * with the called <code>Endpoint</code> and each of the object
109  * identifiers in <code>ids</code> exist for the identified DGC
110  * client.  A clean call asserts that there are no (longer) live
111  * remote references with the called <code>Endpoint</code> and each of
112  * the object identifiers in <code>ids</code> for the identified DGC
113  * client.
114  *
115  * <p>The tracked live remote references are categorized by their
116  * <code>Endpoint</code> and further categorized by their object
117  * identifier (with the <code>Endpoint</code> and object identifier
118  * pair identifying a remote object).  When a new live remote
119  * reference is created, either by construction or deserialization, it
120  * is remembered among the live remote references with the same
121  * <code>Endpoint</code> and object identifier, and its reachability
122  * is tracked with a phantom reference.  If there is not already a
123  * live remote reference with the same <code>Endpoint</code> and
124  * object identifier, the DGC client makes a dirty call to the
125  * server-side DGC implementation at that <code>Endpoint</code>, with
126  * that object identifier in the <code>ids</code> argument.  Dirty
127  * calls for multiple newly created live remote references with the
128  * same <code>Endpoint</code> may be batched as one dirty call (such
129  * as for multiple live remote references deserialized from the same
130  * stream).
131  *
132  * <p>Each successful dirty call establishes or renews a lease for the
133  * DGC client with the server-side DGC implementation at the
134  * <code>Endpoint</code> that the dirty call was made to.  The
135  * duration of the lease granted by the server is conveyed as the
136  * return value of the dirty call, in milliseconds starting from some
137  * time during the processing of the dirty call.  While there are live
138  * remote references with a given <code>Endpoint</code>, the DGC
139  * client attempts to maintain a valid lease with that
140  * <code>Endpoint</code> by renewing its lease with successive dirty
141  * calls.  The DGC client should take into consideration network and
142  * processing latencies of the previous dirty call and the next
143  * required dirty call in choosing when to renew a lease.  If the DGC
144  * client has reason to assume that its lease with a given
145  * <code>Endpoint</code> might have expired, then in subsequent dirty
146  * calls to that <code>Endpoint</code>, it should include the object
147  * identifiers of all currently reachable live remote references with
148  * that <code>Endpoint</code>.  If a dirty call returns a negative
149  * lease duration or throws a {@link NoSuchObjectException}, the DGC
150  * client should refrain from making further dirty calls to the same
151  * <code>Endpoint</code> until a new live remote reference with that
152  * <code>Endpoint</code> is created.
153  *
154  * <p>If a dirty call fails with a communication exception other than
155  * a <code>NonExistantObjectException</code>, the DGC client implementation
156  * should make a reasonable effort to retry the dirty call (in a
157  * network-friendly manner).  Also, after such a failed dirty call for
158  * a given <code>Endpoint</code> and object identifier, any clean call
159  * that is made for that same <code>Endpoint</code> and object
160  * identifier within a reasonable amount of time should pass
161  * <code>true</code> for the <code>strong</code> argument, in case the
162  * failed dirty call does eventually get delivered to the server after
163  * such a clean call has been processed.
164  *
165  * <p>When the last remaining live remote reference with a given
166  * <code>Endpoint</code> and object identifier is detected to be
167  * phantom reachable, the DGC client makes a clean call to the
168  * server-side DGC implementation at that <code>Endpoint</code>, with
169  * that object identifier in the <code>ids</code> argument.  Clean
170  * calls for several object identifiers at the same
171  * <code>Endpoint</code> may be batched as one clean call (such as
172  * when multiple live remote references with the same
173  * <code>Endpoint</code> and different object identifiers are detected
174  * to be phantom reachable at the same time).
175  *
176  * <p>If a clean call fails with a communication exception other than
177  * a <code>NonExistantObjectException</code>, the DGC client implementation
178  * should make a reasonable effort to retry the clean call, in a
179  * network-friendly manner, especially while the DGC client's lease
180  * for the <code>Endpoint</code> remains valid (or while dirty calls
181  * for the same <code>Endpoint</code> succeed).
182  *
183  * @author Sun Microsystems, Inc.
184  * @since 2.0
185  *

191  *
192  * <table summary="Describes what is logged by BasicObjectEndpoint at

201  * activity

203  * </table>

259 	    throw new NullPointerException("null endpoint");
260 	}

263 	}

355     /**
356      * {@inheritDoc}
357      *
358      * <p>This method first invokes {@link Endpoint#newRequest
359      * newRequest} on this <code>BasicObjectEndpoint</code>'s
360      * contained <code>Endpoint</code> with the specified constraints
361      * to obtain an <code>OutboundRequestIterator</code>.  It then
362      * wraps the obtained iterator in another
363      * <code>OutboundRequestIterator</code> and returns the wrapped
364      * iterator.
365      *
366      * <p>The methods of the returned
367      * <code>OutboundRequestIterator</code> behave as follows:
368      *
369      * <p>{@link OutboundRequestIterator#hasNext boolean hasNext()}:
370      *
371      * <blockquote>
372      *
373      * <p>Returns <code>true</code> if this iterator supports making
374      * at least one more attempt to communicate the remote call, and
375      * <code>false</code> otherwise.
376      *
377      * <p>This method invokes <code>hasNext</code> on the underlying
378      * iterator and returns the result.
379      *
380      * <p>The security context in which this method is invoked may be
381      * used for subsequent verification of security permissions; see
382      * the {@link OutboundRequestIterator#next next} method
383      * specification for more details.
384      *
385      * </blockquote>
386      *
387      * <p>{@link OutboundRequestIterator#next OutboundRequest next()}:
388      *
389      * <blockquote>
390      *
391      * <p>Initiates an attempt to communicate the remote call to the
392      * referenced remote object.
393      *
394      * <p>This method invokes <code>next</code> on the underlying
395      * iterator to obtain an <code>OutboundRequest</code>.  Then it
396      * writes the object identifier of the
397      * <code>BasicObjectEndpoint</code> that produced this iterator to
398      * the request's output stream by invoking {@link
399      * Uuid#write(OutputStream) Uuid.write(OutputStream)} with the
400      * request output stream, and then it returns the request.
401      *
402      * <p>Throws {@link NoSuchElementException} if this iterator does
403      * not support making another attempt to communicate the remote
404      * call (that is, if <code>hasNext</code> would return
405      * <code>false</code>).
406      *
407      * <p>Throws {@link IOException} if an <code>IOException</code> is
408      * thrown by the invocation of <code>next</code> on the underlying
409      * iterator or by the subsequent I/O operations.
410      *
411      * <p>Throws {@link SecurityException} if a
412      * <code>SecurityException</code> is thrown by the invocation of
413      * <code>next</code> on the underlying iterator or by the
414      * subsequent I/O operations.
415      *
416      * </blockquote>
417      *
418      * @throws NullPointerException {@inheritDoc}
419      **/
420     public OutboundRequestIterator newCall(InvocationConstraints constraints) {
421 	final OutboundRequestIterator iter = ep.newRequest(constraints);
422 	return new OutboundRequestIterator() {
423 
424 	    public boolean hasNext() {
425 		return iter.hasNext();
426 	    }
427 
428 	    public OutboundRequest next() throws IOException {
429 		OutboundRequest call = iter.next();
430 		boolean ok = false;
431 		try {
432 		    id.write(call.getRequestOutputStream());
433 		    ok = true;
434 		} finally {
435 		    if (!ok) {
436 			call.abort();	// because caller cannot
437 		    }
438 		}
439 		return call;
440 	    }
441 	};
442     }

444     /**
445      * {@inheritDoc}
446      *
447      * <p>This method reads a byte from the response input stream of
448      * <code>call</code>.  If an {@link IOException} is thrown reading
449      * the byte, that exception is thrown to the caller.  If reading
450      * the byte otherwise indicates EOF, an {@link EOFException} is
451      * thrown to the caller.  If the byte is <code>0x00</code>, then
452      * this method returns a {@link NoSuchObjectException} indicating
453      * that there is no remote object exported with the object
454      * identifier at the remote endpoint.  If the byte is
455      * <code>0x01</code>, then this method returns <code>null</code>,
456      * indicating that a remote object corresponding to the object
457      * identifier and endpoint is exported, and thus the caller may
458      * proceed to read the response of the remote call.  If the byte
459      * is any other value, this method returns an {@link
460      * UnmarshalException} indicating that a protocol error occurred.
461      *
462      * @throws NullPointerException {@inheritDoc}
463      **/
464     public RemoteException executeCall(OutboundRequest call)
465 	throws IOException
466     {
467 	// assume that the request output stream has been closed
468 
469 	int status = call.getResponseInputStream().read();
470 	switch (status) {
471 
472 	case -1:
473 	    throw new EOFException();
474 
475 	case 0x00:
476 	    // REMIND: close the response input stream?
477             Exception ex = null;
478             try {
479                 call.getResponseInputStream().close();
480             } catch (IOException e){
481                 ex = e;
482             }
483 	    // REMIND: Do we want to read a server-supplied reason string?
484             if (ex != null){
485                 return new net.jini.export.NonExistantObjectException("no such object in table, input stream close threw IOException: ", ex);
486             }
487 	    return new NoSuchObjectException("no such object in table");
488 
489 	case 0x01:
490 	    return null;
491 
492 	default:
493 	    // REMIND: close the response input stream?
494             Exception exc = null;
495             try {
496                 call.getResponseInputStream().close();
497             } catch (IOException e){
498                 exc = e;
499             }
500 	    // REMIND: Do we really want this failure mode here?
501             if (exc != null){
502                 return new UnmarshalException("unexpected invocation status: " +
503 					  Integer.toHexString(status), exc);
504             }
505 	    return new UnmarshalException("unexpected invocation status: " +
506 					  Integer.toHexString(status));
507 	}
508     }

645      * registered (with the <code>AcknowledgmentSource</code>) that
646      * will hold a strong reference to this
647      * <code>BasicObjectEndpoint</code> until the listener's {@link
648      * net.jini.io.context.AcknowledgmentSource.Listener#acknowledgmentReceived
649      * acknowledgmentReceived} method is invoked (or some other
650      * implementation-specific event occurs, such as a timeout
651      * expiration).
652      * @param out ObjectOutputStream
653      * @throws IOException if a problem occurs during serialization.
654      **/
655     private void writeObject(ObjectOutputStream out) throws IOException {
656 	out.defaultWriteObject();
657 
658 	if (dgc && out instanceof ObjectStreamContext) {
659 	    Collection context =
660 		((ObjectStreamContext) out).getObjectStreamContext();
661 	    for (Iterator i = context.iterator(); i.hasNext();) {
662 		Object e = i.next();
663 		if (e instanceof AcknowledgmentSource) {
664 		    AcknowledgmentSource ackSource = (AcknowledgmentSource) e;
665 		    ackSource.addAcknowledgmentListener(new AckListener(this));
666 		    break;
667 		}
668 	    }
669 	}
670     }

673      * Holds a strong reference to a BasicObjectEndpoint until an
674      * acknowledgment has been received.
675      *
676      * REMIND: Clear reference after a certain timeout regardless?
677      **/
678     private static class AckListener implements AcknowledgmentSource.Listener {
679 	private volatile BasicObjectEndpoint ref;
680 	AckListener(BasicObjectEndpoint ref) { this.ref = ref; }
681 	public void acknowledgmentReceived(boolean received) { ref = null; }
682     }

685      * If this <code>BasicObjectEndpoint</code> participates in DGC,
686      * initiates asynchronous DGC activity for it.
687      *
688      * @throws InvalidObjectException if the <code>Endpoint</code> or
689      * the object identifier is <code>null</code>
690      * @param in ObjectInputStream
691      * @throws ClassNotFoundException if class not found.
692      * @throws IOException if a problem occurs during de-serialization.
693      **/
694     private void readObject(ObjectInputStream in)
695 	throws IOException, ClassNotFoundException
696     {
697 	in.defaultReadObject();
698 	checkSerial(ep, id);
699 	    
700 	/*
701 	 * If DGC-enabled, register this live reference instance to be
702 	 * tracked by the local client-side DGC implementation.  Use
703 	 * an ObjectInputValidation for delayed registration so that
704 	 * multiple live references in the object graph being
705 	 * deserialized can be registered in one batch.
706 	 */
707 	if (dgc) {
708 	    /*
709 	     * REMIND: short circuit lookup with thread local,
710 	     * to avoid synchronization overhead in the common case?
711 	     *
712 	     * Peter Firmstone, 13th December 2014: This synchronization
713 	     * appears to be uncontended in stress tests; there is no 
714 	     * need to optimise at this time.
715 	     */
716 	    DgcBatchContext batchContext;
717 	    synchronized (streamBatches) {
718 		batchContext = (DgcBatchContext) streamBatches.get(in);
719 		if (batchContext == null) {
720 		    batchContext = new DgcBatchContext();
721 		    try {				// REMIND: priority??
722 			in.registerValidation(batchContext, 0);
723 		    } catch (InvalidObjectException e) { // should be NPE
724 			throw new AssertionError();
725 		    }
726 		    streamBatches.put(in, batchContext);
727 		}
728 	    }
729 	    batchContext.addLiveRef(this); // Unsafe publication, final fields haven't been frozen yet.
730 	}
731     }

763     /**
764      * Collects live references to be registered with the local
765      * client-side DGC implementation and registers them in
766      * Endpoint-specific batches.
767      *
768      * REMIND: lack of thread safety OK?
769      **/
770     private static class DgcBatchContext implements ObjectInputValidation {
771 
772 	/* maps Endpoint to List<BasicObjectEndpoint> */
773 	private final Map endpointTable = new HashMap(3);
774 
775 	DgcBatchContext() { }
776 
777 	void addLiveRef(BasicObjectEndpoint ref) {
778 	    /*
779 	     * Organize collected live references into separate lists
780 	     * for each distinct endpoint.
781 	     */
782 	    Endpoint endpoint = ref.getEndpoint();
783 	    Collection refList = (Collection) endpointTable.get(endpoint);
784 	    if (refList == null) {
785 		refList = new ArrayList();
786 		endpointTable.put(endpoint, refList);
787 	    }
788 	    refList.add(ref);
789 	}
790 
791 	public void validateObject() {	// doesn't throw InvalidObjectException
792 	    /*
793 	     * Perform a batch DGC registration for each list of
794 	     * live references to the same endpoint.
795 	     */
796 	    Iterator iter = endpointTable.entrySet().iterator();
797 	    while (iter.hasNext()) {
798 		Map.Entry entry = (Map.Entry) iter.next();
799 		Endpoint endpoint = (Endpoint) entry.getKey();
800 		Collection refList = (Collection) entry.getValue();
801 		dgcClient.registerRefs(endpoint, refList);
802 	    }
803 	    endpointTable.clear();
804 	}
805     }
