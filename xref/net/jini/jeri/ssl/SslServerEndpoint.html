79  /**
80   * An implementation of {@link ServerEndpoint} that uses TLS/SSL to support
81   * invocation constraints for direct communication over TCP sockets. <p>
82   *
83   * Instances of this class are intended to be created for use with the {@link
84   * BasicJeriExporter} class. Calls to {@link #enumerateListenEndpoints
85   * enumerateListenEndpoints} return instances of {@link SslEndpoint}. <p>
86   *
87   * This class supports at least the following constraints, possibly limited by
88   * the available cipher suites: <p>
89   *
90   * <ul>
91   * <li> {@link ClientAuthentication}
92   * <li> {@link ClientMaxPrincipal}, when it contains an {@link X500Principal}
93   * <li> {@link ClientMaxPrincipalType}, when it contains
94   *	<code>X500Principal</code>
95   * <li> {@link ClientMinPrincipal}, when it contains a single
96   *	<code>X500Principal</code> only
97   * <li> {@link ClientMinPrincipalType}, when it contains
98   *	<code>X500Principal</code> only
99   * <li> {@link Confidentiality}
100  * <li> {@link ConfidentialityStrength}, a provider-specific constraint for
101  *	specifying weak or strong confidentiality
102  * <li> {@link ConnectionAbsoluteTime}, trivially, since this only takes effect
103  *	on the client side
104  * <li> {@link ConnectionRelativeTime}, trivially, since this only takes effect
105  *	on the client side
106  * <li> {@link ConstraintAlternatives}, if the elements all have the same
107  *	actual class and at least one element is supported
108  * <li> {@link Delegation#NO}
109  * <li> {@link Delegation#YES}, trivially, for anonymous clients
110  * <li> {@link DelegationAbsoluteTime}, trivially, when delegation is not
111  *	supported
112  * <li> {@link DelegationRelativeTime}, trivially, when delegation is not
113  *	supported
114  * <li> {@link Integrity#YES}
115  * <li> {@link ServerAuthentication}
116  * <li> {@link ServerMinPrincipal}, when it contains a single
117  *	<code>X500Principal</code> only
118  * </ul> <p>
119  *
120  * This class authenticates as a single {@link Principal} if the following
121  * items are present in the server {@link Subject}: <p>
122  *
123  * <ul>
124  * <li> One or more principals of type <code>X500Principal</code>
125  * <li> For each principal, one or more certificate chains, stored as public
126  *      credentials, and represented by instances of {@link CertPath}, whose
127  *      <code>getType</code> method returns "X.509", and for which calling
128  *      <code>getSubjectDN</code> on the certificate chain's first element
129  *      returns that principal's name
130  * <li> For each certificate chain, an instance of {@link
131  *      X500PrivateCredential}, stored as a private credential, whose
132  *      <code>getCertificate</code> method returns a value equal to the first
133  *      element of the certificate chain, and whose <code>getPrivateKey</code>
134  *      method returns the associated private key
135  * </ul> <p>
136  *
137  * In addition, this class will only dispatch remote calls that authenticate as
138  * a given principal if the caller of {@link
139  * net.jini.jeri.ServerEndpoint.ListenEndpoint#listen listen} on the class's
140  * {@link net.jini.jeri.ServerEndpoint.ListenEndpoint} has been granted {@link
141  * AuthenticationPermission} with that principal as the local principal, the
142  * principal representing the authenticated identity of the client for the call
143  * (if any) as the peer principal, and the <code>accept</code> action. <p>
144  *
145  * This class supports remote connections between authenticated servers and
146  * authenticated or anonymous clients, and between anonymous servers and
147  * anonymous clients. Connections between anonymous servers and authenticated
148  * clients are not supported. Because of the suites available in the TLS/SSL
149  * protocol, support for {@link Confidentiality#NO} requires the server to
150  * authenticate with an RSA public key. <p>
151  *
152  * If the server subject contains principals and credentials that would permit
153  * authentication of more than one <code>X500Principal</code>, the endpoint
154  * will make an arbitrary choice of the principal to use for authentication,
155  * and will continue to make the same choice so long as subject contents,
156  * validity of credentials, and security permissions do not change. <p>
157  *
158  * The host name specified when creating an <code>SslServerEndpoint</code>
159  * instance controls the host name that will be contained in
160  * <code>SslEndpoint</code> instances produced when {@link
161  * #enumerateListenEndpoints enumerateListenEndpoints} is invoked to listen on
162  * the server endpoint; the host name does not affect the behavior of the
163  * listen operation itself, which listens on all of the local system's network
164  * addresses. If the host name in the server endpoint is <code>null</code>,
165  * then the host name in the <code>SslEndpoint</code> instances that it
166  * produces will be the default server host name, which is the IP address
167  * string of the {@link InetAddress} returned by {@link
168  * InetAddress#getLocalHost InetAddress.getLocalHost} when
169  * <code>enumerateListenEndpoints</code> is invoked. <p>
170  *
171  * This class permits specifying a {@link SocketFactory} for creating the
172  * {@link Socket} instances that the associated <code>SslEndpoint</code>
173  * instances use to make remote connections back to the server, and a {@link
174  * ServerSocketFactory} for creating the {@link ServerSocket} instances that
175  * the server endpoint uses to accept remote connections. These socket
176  * factories and sockets should not implement the TLS/SSL protocol; it is the
177  * responsibility of the implementation to establish TLS/SSL connections over
178  * the sockets it obtains from the socket factories. In particular, instances
179  * of {@link SSLSocketFactory} and {@link SSLServerSocketFactory} should not be
180  * used, and the factories used should not return instances of {@link
181  * SSLSocket} or {@link SSLServerSocket}. <p>
182  *
183  * A <code>SocketFactory</code> used with instances of this class should be
184  * serializable, and must implement {@link Object#equals Object.equals} to obey
185  * the guidelines that are specified for <code>equals</code> methods of {@link
186  * Endpoint} instances. A <code>ServerSocketFactory</code> used with instances
187  * of this class must implement <code>Object.equals</code> to obey the
188  * guidelines that are specified for <code>equals</code> methods of {@link
189  * net.jini.jeri.ServerEndpoint.ListenEndpoint ListenEndpoint} instances. <p>
190  *
191  * This class uses the <a href="../connection/doc-files/mux.html">Jini
192  * extensible remote invocation (Jini ERI) multiplexing protocol</a> to map
193  * outgoing requests to socket connections.
194  *
195  * 
196  * @see SslEndpoint
197  * @see ConfidentialityStrength
198  * @since 2.0
199  *

205  *

259  *
260  * This implementation uses the following security providers: <p>
261  *
262  * <ul>
263  * <li> {@link SSLContext}, with the protocol specified by the
264  *	<code>org.apache.river.jeri.ssl.sslProtocol</code> system property, or
265  *	<code>"TLS"</code> if that property is not defined, to provide the
266  *	TLS/SSL implementation. The {@link SSLContext#init SSLContext.init}
267  *	method is called with <code>null</code> for the <code>random</code>
268  *	parameter to use the default {@link SecureRandom} implementation.
269  * <li> {@link CertificateFactory}, with type <code>"X.509"</code>, to generate
270  *	<code>CertPath</code> instances from X.509 certificate chains
271  * <li> {@link TrustManagerFactory}, with the algorithm specified by the
272  *	<code>org.apache.river.jeri.ssl.trustManagerFactoryAlgorithm</code> system
273  *	property, or the default algorithm if that property is not defined, to
274  *	implement trust management for the TLS/SSL implementation. The factory
275  *	must return trust managers that implement {@link X509TrustManager}.
276  * </ul> <p>
277  * 
278  * See the documentation on <a
279  * href="http://java.sun.com/j2se/1.4/docs/guide/security/CryptoSpec.html#ProviderInstalling"
280  * target="_top">installing security providers</a> and <a
281  * href="http://java.sun.com/j2se/1.4/docs/guide/security/jsse/JSSERefGuide.html#ProviderCust"
282  * target="_top">configuring JSSE</a> for information on configuring these
283  * providers. <p>
284  *
285  * The <a
286  * href="http://java.sun.com/j2se/1.4/docs/guide/security/jsse/JSSERefGuide.html#Customization"
287  * target="_top">JSSE documentation</a> also describes the system properties
288  * for configuring the location, type, and password of the truststore that this
289  * implementation uses, through JSSE, to make decisions about what certificate
290  * chains should be trusted. <p>
291  * 
292  * This implementation recognizes the following system properties: <p>
293  * 
294  * <ul>
295  * <li> <code>org.apache.river.jeri.ssl.maxServerSessionDuration</code> - The
296  *	maximum number of milliseconds a server-side TLS/SSL session should be
297  *	used before expiring. The default is 24 hours. The value used should be
298  *	larger than the maximum client session duration to allow the client to
299  *	negotiate a new session before the server timeout occurs.
300  * <li> <code>org.apache.river.jeri.ssl.sslProtocol</code> - The secure socket
301  *	protocol used when obtaining {@link SSLContext} instances. The default
302  *	is <code>"TLS"</code>.
303  * <li> <code>org.apache.river.jeri.ssl.trustManagerFactoryAlgorithm</code> - The
304  *	algorithm used when obtaining {@link TrustManagerFactory}
305  *	instances. The default is the value returned by {@link
306  *	TrustManagerFactory#getDefaultAlgorithm
307  *	TrustManagerFactory.getDefaultAlgorithm}.
308  * <li> <code>org.apache.river.jeri.ssl.cipherSuites</code> - The TLS/SSL cipher
309  *	suites that should be used for communication. The default is the list
310  *	of suites supported by the JSSE implementation. The value should
311  *	specify the suite names, separated by commas. The value will be ignored
312  *	if it contains no suites or specifies suites that are not supported by
313  *	the JSSE implementation. Suites appearing earlier in the list will be
314  *	preferred to ones appearing later for suites that support the same
315  *	requirements and preferences.
316  * </ul>
