36  /**
37   * The <code>JavaSpace05</code> interface extends the {@link
38   * JavaSpace} interface to provide methods that allow clients to
39   * perform batch operations on the space (which are generally more
40   * efficient than performing an equivalent set of singleton
41   * operations) and to more easily develop applications where a
42   * given {@link Entry} needs to be read by multiple clients.
43   * Implementations of the <code>JavaSpace</code> interface are not
44   * required to implement this interface.<p>
45   *
46   * Unless otherwise noted, the effects of any invocation of a
47   * method defined by this interface must be visible to any
48   * operation on the space that is started after the invocation
49   * returns normally. Note, the effects of a method invocation that
50   * throws a {@link RemoteException} are not necessarily visible
51   * when the exception is thrown. <p>
52   *
53   * All of the methods of this interface take one or more {@link
54   * Collection}s as arguments. Each such <code>Collection</code>
55   * must be treated as immutable by implementations and must not be
56   * changed by the client during the course of any method
57   * invocation to which they have been passed.<p>
58   *
59   * This interface is not a remote interface. Each implementation of
60   * this interface exports a proxy object that implements this
61   * interface local to the client.  Each method of the interface
62   * takes as one of its arguments a <code>Collection</code> of
63   * <code>Entry</code> instances. The entries themselves must be
64   * serialized in accordance with the <a
65   * href=http://www.jini.org/standards/index.html>Jini Entry
66   * Specification</a> and will not be altered by the
67   * call. Typically, the <code>Collection</code> holding the entries
68   * will not be serialized at all. If one of these entries can't be
69   * serialized, a {@link MarshalException} will be thrown. Aside
70   * from the handling of these <code>Collection</code> of
71   * <code>Entry</code> parameters, all methods defined by this
72   * interface otherwise obey normal Java(TM) Remote Method
73   * Invocation remote interface semantics.
74   *
75   * @see <a href=http://www.jini.org/standards/index.html>;
76   *      JavaSpaces Service Specification</a>
77   * @see <a href=http://www.jini.org/standards/index.html>;
78   *      Jini Entry Specification</a>
79   * @since 2.1
80   */
81  public interface JavaSpace05 extends JavaSpace {
82      /**
83       * This method provides an overload of the {@link
84       * JavaSpace#write JavaSpace.write} method that allows new
85       * copies of multiple {@link Entry} instances to be stored in
86       * the space using a single call. The client may specify a
87       * {@link Transaction} for the operation to be performed
88       * under. Each <code>Entry</code> to be stored in the space
89       * has a separate requested initial lease duration. <p>
90       *
91       * The effect on the space of an invocation of this method
92       * successfully storing an <code>Entry</code> is the same as if
93       * the <code>Entry</code> had been successfully stored by a
94       * call to the singleton form of <code>write</code> under
95       * <code>txn</code> with the given requested initial lease
96       * duration. This method returns the proxies to the leases for
97       * each newly stored <code>Entry</code> by returning a {@link
98       * List} of {@link Lease} instances. The <em>i</em> th element
99       * of the returned <code>List</code> will be a proxy for the
100      * lease on the <code>Entry</code> created from the <em>i</em>
101      * th element of <code>entries</code>. <p>
102      *
103      * If an invocation of this method returns normally, then a new
104      * copy of each element of <code>entries</code> must have been
105      * stored in the space. A new copy of each element will be
106      * stored even if there are duplicates (either in terms of
107      * object identity or of entry equivalence) in
108      * <code>entries</code>. <p>
109      * 
110      * The order in which the entries stored by an invocation of
111      * this method will become visible in the space is unspecified,
112      * and different observers may see the entries become visible
113      * in different orders.<p>
114      *
115      * If a {@link TransactionException}, {@link
116      * SecurityException}, {@link IllegalArgumentException}, or
117      * {@link NullPointerException} is thrown, no entries will
118      * have been added to the space by this operation. If a {@link
119      * RemoteException} is thrown, either new copies of all of the
120      * elements of <code>entries</code> will have been stored or
121      * no entries will have been stored; that is, in the case of a
122      * <code>RemoteException</code>, the storing of new entries in
123      * the space will either fail or succeed as a unit. <p>
124      *
125      * @param entries a {@link List} of {@link Entry} instances to
126      *                be written to the space
127      * @param txn the {@link Transaction} this operation should be
128      *                performed under, may be <code>null</code>
129      * @param leaseDurations a <code>List</code> of
130      *                {@link Long}s representing the
131      *                requested initial lease durations
132      * @return a <code>List</code> of {@link Lease} instances, one
133      *         for each element of <code>entries</code>, may be
134      *         immutable. The space will not keep a reference to
135      *         the result
136      * @throws TransactionException if <code>txn</code> is
137      *         non-<code>null</code> and is not usable by the
138      *         space
139      * @throws RemoteException if a communication error occurs
140      * @throws IllegalArgumentException if <code>entries</code> and
141      *         <code>leaseDurations</code> are not the same length
142      *         or are empty, if any element of <code>entries</code>
143      *         is not an instance of <code>Entry</code>, if any
144      *         element of <code>leaseDurations</code> is not an
145      *         instance of <code>Long</code>, or if any element of
146      *         <code>leaseDurations</code> is a negative value
147      *         other than {@link Lease#ANY Lease.ANY}
148      * @throws NullPointerException if either <code>entries</code>
149      *         or <code>leaseDurations</code> is <code>null</code>
150      *         or contains a <code>null</code> value
151      */
152     public List write(List        entries,
153 		      Transaction txn,
154 		      List        leaseDurations)
155 	throws TransactionException, RemoteException;

157     /**
158      * This method provides an overload of the {@link
159      * JavaSpace#take JavaSpace.take} method that attempts to
160      * remove, optionally under a {@link Transaction}, and return
161      * one or more entries from the space. Each {@link Entry}
162      * taken will match one or more elements of the passed {@link
163      * Collection} of templates, and all of the taken entries will
164      * be visible to the passed <code>Transaction</code>. If there
165      * are initially no matches in the space that are visible to
166      * the passed <code>Transaction</code>, an invocation of this
167      * method will block for up to a specified timeout for one or
168      * more matches to appear. <p>
169      *
170      * The effect on the space of an invocation of this method
171      * successfully taking an <code>Entry</code> will be the same
172      * as if the <code>Entry</code> had been taken using the
173      * singleton version of this method and passing
174      * <code>txn</code> as the <code>Transaction</code>. <p>
175      *
176      * The <code>tmpls</code> parameter must be a
177      * <code>Collection</code> of <code>Entry</code> instances to
178      * be used as templates. All of the entries taken must match
179      * one or more of these templates. The <code>tmpls</code>
180      * parameter may contain <code>null</code> values and may
181      * contain duplicates. An <code>Entry</code> is said to be
182      * <em>available</em> to an invocation of this method if the
183      * <code>Entry</code> could have been returned by an
184      * invocation of the singleton <code>take</code> method using
185      * <code>txn</code>. <p>
186      *
187      * If the method succeeds, a non-<code>null</code>
188      * <code>Collection</code> will be returned. The
189      * <code>Collection</code> will contain a copy of each
190      * <code>Entry</code> that was taken. If no entries were taken,
191      * the <code>Collection</code> will be empty.  Each
192      * <code>Entry</code> taken will be represented by a distinct
193      * <code>Entry</code> instance in the returned
194      * <code>Collection</code>, even if some of the entries are
195      * equivalent to others taken by the operation. There will be
196      * no <code>null</code> elements in the returned 
197      * <code>Collection</code>.<p>
198      *
199      * If one or more of the entries taken cannot be unmarshalled
200      * in the client, an {@link UnusableEntriesException} is
201      * thrown. The exception's {@link
202      * UnusableEntriesException#getEntries
203      * UnusableEntriesException.getEntries} method will return a
204      * <code>Collection</code> with a copy of each
205      * <code>Entry</code> that could be unmarshalled. The {@link
206      * UnusableEntriesException#getUnusableEntryExceptions
207      * UnusableEntriesException.getUnusableEntryExceptions} method
208      * will return a <code>Collection</code> with an {@link
209      * UnusableEntryException} for each <code>Entry</code> that
210      * could not be unmarshalled. Every <code>Entry</code> taken
211      * by the invocation will either be represented in the
212      * <code>Collection</code> returned by <code>getEntries</code>
213      * or in the <code>Collection</code> returned by
214      * <code>getUnusableEntryExceptions</code>. <p>
215      *
216      * If there is at least one matching <code>Entry</code>
217      * available in the space, an invocation of this method must
218      * take at least one <code>Entry</code>. If more than one
219      * matching <code>Entry</code> is available, the invocation may
220      * take additional entries. It must not take more than
221      * <code>maxEntries</code>, but an implementation may chose to
222      * take fewer entries from the space than the maximum available or
223      * the maximum allowed by <code>maxEntries</code>.  If for
224      * whatever reason, an invocation of this method takes fewer
225      * entries than the maximum number of available matching
226      * entries, how an implementation selects which entries should be
227      * taken by the invocation and which are left in the space is
228      * unspecified. How consumption of entries is arbitrated
229      * between conflicting queries is also unspecified. <p>
230      *
231      * If there are initially no matching entries in the space, an
232      * invocation of this method should block for up to
233      * <code>timeout</code> milliseconds for a match to appear. If
234      * one or more matches become available before
235      * <code>timeout</code> expires, one or more of the newly
236      * available entries should be taken and the method should
237      * return without further blocking. If for some reason the
238      * invocation can't block for the full timeout and no entries
239      * have been taken, the invocation must fail with a {@link
240      * RemoteException} or {@link TransactionException} as
241      * appropriate. <p>
242      *
243      * If an invocation of this method removes (or locks) more than
244      * one <code>Entry</code>, the order in which the removal (or
245      * locking) occurs is undefined, and different observers may see
246      * the removal or locking of the entries in different
247      * orders. <p>
248      *
249      * If a <code>TransactionException</code>, {@link
250      * SecurityException}, {@link IllegalArgumentException}, or
251      * {@link NullPointerException} is thrown, no entries will
252      * have been taken. If a <code>RemoteException</code> is
253      * thrown, up to <code>maxEntries</code> may have been taken
254      * by this operation. <p>
255      *
256      * @param tmpls a {@link Collection} of {@link Entry}
257      *              instances, each representing a template. All
258      *              of the entries taken by an invocation of this
259      *              method will match one or more elements of
260      *              <code>tmpls</code>
261      * @param txn the {@link Transaction} this operation should be
262      *              performed under, may be <code>null</code>
263      * @param timeout if there are initially no available
264      *              matches in the space, the maximum number of
265      *              milliseconds to block waiting for a match to
266      *              become available
267      * @param maxEntries the maximum number of entries that may be
268      *              taken by this method
269      * @return a <code>Collection</code> that contains a copy of each
270      *         <code>Entry</code> taken from the space by this
271      *         method. The space will not keep a reference to
272      *         this <code>Collection</code>. May be immutable
273      * @throws UnusableEntriesException if one or more of the
274      *         entries taken can't be unmarshalled in the client
275      * @throws TransactionException if <code>txn</code> is
276      *         non-<code>null</code> and is not usable by the
277      *         space.
278      * @throws RemoteException if a communication error occurs
279      * @throws IllegalArgumentException if any non-<code>null</code>
280      *         element of <code>tmpls</code> is not an instance of
281      *         <code>Entry</code>, if <code>tmpls</code> is empty,
282      *         if <code>timeout</code> is negative, or if
283      *         <code>maxEntries</code> is non-positive
284      * @throws NullPointerException if <code>tmpls</code> is null
285      */
286     public Collection take(Collection  tmpls, 
287 			   Transaction txn, 
288 			   long        timeout, 
289 			   long        maxEntries)
290 	throws UnusableEntriesException, TransactionException,
291 	       RemoteException;

293     /**
294      * Creates a {@linkplain MatchSet match set} that can be used to
295      * exhaustively read through all of the matching entries in
296      * the space that are visible to the passed {@link
297      * Transaction} and remain visible for the lifetime of the
298      * match set. May also yield additional entries that match but
299      * are only visible for part of the lifetime of the match
300      * set. <p>
301      *
302      * The <code>tmpls</code> parameter must be a {@link
303      * Collection} of {@link Entry} instances to be used as
304      * templates. All of the entries placed in the match set will
305      * match one or more of these templates. <code>tmpls</code> may
306      * contain <code>null</code> values and may contain
307      * duplicates. An <code>Entry</code> is said to be
308      * <em>visible</em> to an invocation of this method if the
309      * <code>Entry</code> could have been returned by a singleton
310      * {@link JavaSpace#read JavaSpace.read} using the same
311      * transaction. <p>
312      *
313      * The resulting match set must initially contain all of the
314      * visible matching entries in the space. During the lifetime
315      * of the match set an <code>Entry</code> may be, but is not
316      * required to be, added to the match set if it becomes
317      * visible. If the match set becomes empty, no more entries can
318      * be added and the match set enters the {@linkplain MatchSet
319      * exhausted} state.<p>
320      * 
321      * Normally there are three conditions under which an
322      * <code>Entry</code> might be removed from the match set:
323      * 
324      * <ul>
325      *
326      * <li> Any <code>Entry</code> yielded by an invocation of the
327      * {@link MatchSet#next MatchSet.next} method on the match
328      * set (either as the return value of a successful call or
329      * embedded in an {@link UnusableEntryException}) must be
330      * removed from the match set.
331      *
332      * <li> Any <code>Entry</code> that remains in the match set
333      * after <code>maxEntries</code> entries are yielded by
334      * <code>next</code> invocations must be removed from the
335      * match set. In such a case, the criteria used to select which
336      * entries are yielded by <code>next</code> calls and which
337      * get removed from the set at the end is unspecified.
338      * 
339      * <li> Any <code>Entry</code> that during the lifetime of the
340      * match set becomes invisible may at the discretion of the
341      * implementation be removed from the match set.
342      *
343      * </ul>
344      * <p>
345      *
346      * An implementation may decide to remove an <code>Entry</code>
347      * from the set for other reasons.  If it does so, however, it
348      * must {@linkplain MatchSet invalidate} the set.<p>
349      *
350      * If <code>txn</code> is non-<code>null</code> and still
351      * active, any <code>Entry</code> removed from the match set by
352      * a <code>next</code> call must be locked as if it had been
353      * returned by a read operation using <code>txn</code>. An
354      * implementation may establish the read lock on the
355      * <code>Entry</code> any time between when the
356      * <code>Entry</code> is added to the match set and when the
357      * <code>Entry</code> is removed from the match set by an
358      * invocation of <code>next</code>. These read locks are not
359      * released when the match set reaches either the exhausted
360      * state or the invalidated state. If from the space's
361      * perspective the <code>txn</code> leaves the active state,
362      * the space must remove from the match set any entries in the
363      * match set that have not yet been read locked. This may
364      * require the match set to be invalidated. <p>
365      * 
366      * If the match set is leased and <code>leaseDuration</code>
367      * is positive, the initial duration of the lease must be less
368      * than or equal to <code>leaseDuration</code>. If
369      * <code>leaseDuration</code> is {@link Lease#ANY Lease.ANY},
370      * the initial duration of the lease can be any positive value
371      * desired by the implementation. <p>
372      *
373      * If there are {@linkplain net.jini.core.constraint remote
374      * method constraints} associated with an invocation of this
375      * method, any remote communications performed by or on behalf
376      * of the match set's <code>next</code> method will be
377      * performed in compliance with these constraints, not with the
378      * constraints (if any) associated with <code>next</code>. <p>
379      *
380      * @param tmpls a {@link Collection} of {@link Entry}
381      *              instances, each representing a template. All
382      *              the entries added to the resulting match set will
383      *              match one or more elements of <code>tmpls</code>
384      * @param txn   the {@link Transaction} this operation should be
385      *              performed under, may be <code>null</code>
386      * @param leaseDuration the requested initial lease time on
387      *              the resulting match set
388      * @param maxEntries the maximum number of entries to remove
389      *              from the set via {@link MatchSet#next MatchSet.next} 
390      *              calls
391      * @return A proxy to the newly created {@linkplain MatchSet match set}
392      * @throws TransactionException if <code>txn</code> is
393      *         non-<code>null</code> and is not usable by the
394      *         space
395      * @throws RemoteException if a communication error occurs
396      * @throws IllegalArgumentException if any non-<code>null</code>
397      *         element of <code>tmpls</code> is not an instance of
398      *         <code>Entry</code>, if <code>tmpls</code> is empty,
399      *         if <code>leaseDuration</code> is neither positive
400      *         nor {@link Lease#ANY Lease.ANY}, or if
401      *         <code>maxEntries</code> is non-positive
402      * @throws NullPointerException if <code>tmpls</code> is
403      *         <code>null</code> 
404      */
405     public MatchSet contents(Collection  tmpls, 
406 			     Transaction txn,
407 			     long        leaseDuration, 
408 			     long        maxEntries)
409 	throws TransactionException, RemoteException;

411     /**
412      * Register for events triggered when a matching {@link Entry}
413      * transitions from unavailable to available.  The resulting
414      * events will be instances of the {@link AvailabilityEvent}
415      * class and the {@link AvailabilityEvent#getEntry
416      * AvailabilityEvent.getEntry} method will return a copy of
417      * the <code>Entry</code> whose transition triggered the
418      * event.<p>
419      *
420      * An <code>Entry</code> makes a transition from
421      * <em>unavailable to available</em> when it goes from being in
422      * a state where it could not be returned by a {@link
423      * JavaSpace#take JavaSpace.take} using <code>txn</code> to a
424      * state where it could be returned. An <code>Entry</code>
425      * makes a transition from <em>invisible to visible</em> when
426      * it goes from being in a state where it could not be returned
427      * by a {@link JavaSpace#read JavaSpace.read} using
428      * <code>txn</code> to a state where it could be
429      * returned. Note, any transition from invisible to visible is
430      * also a transition from unavailable to available, but an
431      * already visible entry can be unavailable and then make a
432      * transition from unavailable to available. Because the entry
433      * was already visible, this transition would not be a
434      * transition from invisible to visible.<p>
435      *
436      * The <code>tmpls</code> parameter must be a {@link
437      * Collection} of <code>Entry</code> instances to be used as
438      * templates. Events will be generated when an
439      * <code>Entry</code> that matches one or more of these
440      * templates makes an appropriate transition. A single
441      * transition will generate only one event per registration, in
442      * particular the transition of an <code>Entry</code> that
443      * matches multiple elements of <code>tmpls</code> must still
444      * generate exactly one event for this registration. If a given
445      * <code>Entry</code> undergoes multiple applicable transitions
446      * while the registration is active, each must generate a
447      * separate event.<p>
448      * 
449      * Events are not generated directly by the transition of
450      * matching entries, but instead by an abstract observer set up
451      * in the space for each registration. The observer may see the
452      * transitions out of order and as a result the order of the
453      * events generated for this registration (as determined by the
454      * sequence numbers assigned to the events) may be different
455      * from the order of the transitions themselves. Additionally,
456      * each registration will have its own abstract observer and
457      * different observers may see the same sequence of transitions
458      * in different orders. As a result, given a set of transitions
459      * that trigger events for two different registrations, the
460      * order of the events generated for one registration may
461      * differ from the order of the events generated for the
462      * other. <p>
463      *
464      * A non-<code>null</code> {@link EventRegistration} object
465      * will be returned.  Each registration will be assigned an
466      * event ID. The event ID will be unique at least with respect
467      * to all other active event registrations for
468      * <code>AvailabilityEvent</code>s on this space with a
469      * non-equivalent set of templates, a different transaction,
470      * and/or a different value for the
471      * <code>visibilityOnly</code> flag. The event ID can be
472      * obtained by calling the {@link EventRegistration#getID
473      * EventRegistration.getID} method on the returned
474      * <code>EventRegistration</code>. The returned
475      * <code>EventRegistration</code> object's {@link
476      * EventRegistration#getSource EventRegistration.getSource}
477      * method will return a reference to the space.<p>
478      * 
479      * Registrations are leased. <code>leaseDurations</code>
480      * represents the client's desired initial lease duration.  If
481      * <code>leaseDuration</code> is positive, the initial lease
482      * duration will be a positive value less than or equal to
483      * <code>leaseDuration</code>. If <code>leaseDuration</code> is
484      * {@link Lease#ANY Lease.ANY}, the space is free to pick any
485      * positive initial lease duration it desires. A proxy for the
486      * lease associated with the registration can be obtained by
487      * calling the returned <code>EventRegistration</code>'s {@link
488      * EventRegistration#getLease EventRegistration.getLease}
489      * method.<p>
490      *
491      * A registration made with a non-<code>null</code> value for
492      * <code>txn</code> is implicitly dropped when the space
493      * observes <code>txn</code> has left the active state.<p>
494      *
495      * @param tmpls a {@link Collection} of {@link Entry}
496      *              instances, each representing a
497      *              template. Events for this registration will be
498      *              generated by the transitions of entries
499      *              matching one or more elements of
500      *              <code>tmpls</code>
501      * @param txn   the {@link Transaction} this operation should be
502      *              performed under, may be <code>null</code>
503      * @param visibilityOnly if <code>true</code>, events will
504      *              be generated for this registration only when a
505      *              matching <code>Entry</code> transitions from
506      *              invisible to visible, otherwise events will be
507      *              generated when a matching <code>Entry</code>
508      *              makes any transition from unavailable to
509      *              available
510      * @param listener the object to which events generated for
511      *              this registration should be delivered
512      * @param leaseDuration the requested initial lease time on
513      *              the resulting event registration
514      * @param handback the {@link MarshalledObject} to be 
515      *              returned by the {@link
516      *              RemoteEvent#getRegistrationObject 
517      *              RemoteEvent.getRegistrationObject} method of
518      *              the events generated for this registration
519      * @return an {@link EventRegistration} object with
520      *         information on this registration
521      * @throws TransactionException if <code>txn</code> is
522      *         non-<code>null</code> and is not usable by the
523      *         space
524      * @throws RemoteException if a communication error occurs
525      * @throws IllegalArgumentException if any non-<code>null</code>
526      *         element of <code>tmpls</code> is not an instance of
527      *         <code>Entry</code>, if <code>tmpls</code> is empty,
528      *         or if <code>leaseDuration</code> is neither
529      *         positive nor {@link Lease#ANY Lease.ANY}
