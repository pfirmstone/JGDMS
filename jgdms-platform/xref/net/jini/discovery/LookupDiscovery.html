28  /**
29   * This class is a helper utility class that encapsulates the functionality 
30   * required of an entity that wishes to employ multicast discovery to 
31   * find lookup services located within the entity's "multicast radius" 
32   * (roughly, the number of hops beyond which neither the multicast requests
33   * from the entity, nor the multicast announcements from the lookup service,
34   * will propagate). This class helps make the process of acquiring references
35   * to lookup services - based on no information other than lookup service
36   * group membership - much simpler for both services and clients.
37   *
38   *  <!-- Implementation Specifics -->
39   *
40   * The following implementation-specific items are discussed below:
41   * <ul><li> <a href="#ldConfigEntries">Configuring LookupDiscovery</a>
42   *     <li> <a href="#ldLogging">Logging</a>
43   * </ul>
44   *
45   * <a name="ldConfigEntries"><b>Configuring LookupDiscovery</b></a>
46   * 
47   * This implementation of <code>LookupDiscovery</code> supports the
48   * following configuration entries; where each configuration entry name
49   * is associated with the component name
50   * <code>net.jini.discovery.LookupDiscovery</code>. Note that the
51   * configuration entries specified here are specific to this implementation
52   * of <code>LookupDiscovery</code>. Unless otherwise stated, each entry
53   * is retrieved from the configuration only once per instance of this
54   * utility, where each such retrieval is performed in the constructor.
55   *
56   * <a name="discoveryConstraints"></a>
57   * <table summary="Describes the discoveryConstraints
58   *                configuration entry" border="0" cellpadding="2">
59   *   <tr valign="top">
60   *     <th scope="col">&#X2022;
61   *     <th scope="col" align="left" colspan="2">
62   *     <code>discoveryConstraints</code>
63   * 
64   *   <tr valign="top"> <td> &nbsp; <th scope="row" align="right">
65   *     Type: <td> {@link net.jini.core.constraint.MethodConstraints}
66   * 
67   *   <tr valign="top"> <td> &nbsp; <th scope="row" align="right">
68   *     Default: <td> <code>null</code>
69   * 
70   *   <tr valign="top"> <td> &nbsp; <th scope="row" align="right">
71   *     Description:
72   *       <td> Constraints to apply to the multicast request, multicast
73   *         	announcement and unicast discovery protocols.  Multicast
74   *         	request constraints are derived by calling
75   *         	{@link MethodConstraints#getConstraints getConstraints} on the
76   *         	obtained <code>MethodConstraints</code> instance with a
77   *         	<code>Method</code> object for the
78   *    {@link org.apache.river.discovery.DiscoveryConstraints#multicastRequest
79   *              multicastRequest} method; multicast announcement and unicast
80   *              discovery constraints are similarly obtained by passing
81   *              <code>Method</code> objects for the
82   *    {@link org.apache.river.discovery.DiscoveryConstraints#multicastAnnouncement
83   *              multicastAnnouncement} and
84   *    {@link org.apache.river.discovery.DiscoveryConstraints#unicastDiscovery
85   *              unicastDiscovery} methods, respectively.  A <code>null</code>
86   *              value is interpreted as	mapping all methods to empty
87   *              constraints.
88   *         	<p>
89   *         	This class supports the use of the following constraint types
90   *              to control discovery behavior:
91   *         	<ul>
92   *         	  <li> {@link org.apache.river.discovery.DiscoveryProtocolVersion}:
93   *         	       this constraint can be used to control which version(s)
94   *         	       of the multicast request, multicast announcement and
95   *         	       unicast discovery protocols are used.
96   *         	  <li> {@link org.apache.river.discovery.MulticastMaxPacketSize}:
97   *         	       this constraint can be used to control the maximum size
98   *         	       of multicast request packets to send; it can also be
99   *         	       used to specify the size of the buffer used to receive
100  *         	       incoming multicast announcement packets.
101  *         	  <li> {@link org.apache.river.discovery.MulticastTimeToLive}: this
102  *         	       constraint can be used to control the time to live (TTL)
103  *         	       value set on outgoing multicast request packets.
104  *         	  <li> {@link org.apache.river.discovery.UnicastSocketTimeout}:
105  *         	       this constraint can be used to control the read timeout
106  *         	       set on sockets over which unicast discovery is
107  *         	       performed.
108  *         	  <li> {@link net.jini.core.constraint.ConnectionRelativeTime}:
109  *         	       this constraint can be used to control the relative
110  *                     connection timeout set on sockets over which unicast
111  *                     discovery is performed.
112  *          	  <li> {@link net.jini.core.constraint.ConnectionAbsoluteTime}:
113  *         	       this constraint can be used to control the absolute
114  *                     connection timeout set on sockets over which unicast
115  *                     discovery is performed.
116  *         	</ul>
117  *         	Constraints other than those listed above are passed on to the
118  *         	underlying implementations of versions 1 and 2 of the discovery
119  *              protocols.
120  * </table>
