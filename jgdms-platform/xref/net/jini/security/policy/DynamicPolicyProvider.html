55  /**
56   * Security policy provider that supports dynamic granting of permissions at
57   * run-time.  This provider is designed as a wrapper to layer dynamic grant
58   * functionality on top of an underlying policy provider.  If the underlying
59   * provider does not implement the {@link DynamicPolicy} interface, then its
60   * permission mappings are assumed to change only when its {@link
61   * Policy#refresh refresh} method is called.  Permissions are granted on the
62   * granularity of class loader; granting a permission requires (of the calling
63   * context) {@link GrantPermission} for that permission.
64   * 
65   * <p>This is a Dynamic Policy Provider that supports concurrent access,
66   * for instances where a Policy provider is used for a distributed network
67   * of computers, or where there is a large number of ProtectionDomains and
68   * hence the opportunity for concurrency exists, concurrency comes with a 
69   * cost however, that of increased memory usage.</p>
70   * 
71   * <p>Due to the Java 2 Security system's static design, a Policy Provider
72   * can only augment policy files; a Policy can only relax security
73   * by granting additional permissions, this implementation adds an experimental 
74   * feature for revoking permissions, however there are some caveats:</p>
75   * 
76   * <p>A ProtectionDomain must
77   * be created with the dynamic constructor otherwise it will never consult
78   * the policy.  The AccessController.checkPermission(Permission) method
79   * consults the current AccessControlContext, which contains all
80   * ProtectionDomain's on the current thread's stack, before consulting the
81   * AccessControllContext.checkPermission(Permission), it calls
82   * AccessControllContext.optimize() which  removes all duplicate ProtectionDomains
83   * in the ProtectionDomain array[]'s from the
84   * enclosing AccessContolContext for the execution domain and the nested
85   * AccessControlContext for the privileged domain (the privileged domain is
86   * an array of ProtectionDomain's on the stack since the last 
87   * AccessController.doPriveleged() call).  The optimize() method also calls
88   * the DomainCombiner, which, for example, gives the SubjectDomainCombiner the 
89   * opportunity to manipulate the ProtectionDomain's in the privileged array, in the
90   * SubjectDomainCombiner's case, it creates new copies of the ProtectionDomain's
91   * with new Principal[]'s injected.  The optimize() method returns a new
92   * optimized AccessControlContext.
93   * </p><p>
94   * Now the AccessController calls the new AccessControlContext.checkPermission(Permission),
95   * at this stage, each ProtectionDomain, if created with the dynamic constructor
96   * consults the Policy, calling Policy.implies(ProtectionDomain, Permission).
97   * </p><p>
98   * If any calls to the policy return false, the ProtectionDomain then checks its
99   * internal Permissions and if they return false, it returns false.  The first
100  * ProtectionDomain in the AccessControlContext to return false causes the 
101  * AccessController.checkPermission(Permission) to throw an AccessControlException
102  * </p><p>
103  * To optimise the time taken to check Permission's the ProtectionDomain's
104  * should either be static, which excludes the Policy, or dynamic with
105  * a null PermissionCollection in it's constructor, </p>
106  * 
107  * <p>So in order to prevent dynamic grants from finding
108  * their way into a ProtectionDomain's private PermissionCollection,
109  * this policy ensures that no dynamically grantable permissions are 
110  * returned via the method:</p>
111  * <p>
112  * getPermissions(Codesource source) as a precaution.
113  * </p>
114  * <p>This is different to the behaviour of the previous Jini 2.0
115  * DynamicPolicyProvider implementation where dynamically granted Permissions
116  * could escape into the ProtectionDomain's private PermissionCollection.
117  * </p>
118  * <p>
119  * It is thus recommended that Static policy files only be used for setting
120  * up your privileged code and use UmbrellaGrantPermission's and grant 
121  * all other Permission's using dynamic grants. 
122  *
123  * </p><p>
124  * The underlying policy provider may be set using the System property:
125  * </p>,<p>
126  * net.jini.security.policy.PolicyFileProvider.basePolicyClass = 
127  * org.apache.river.security.concurrent.ConcurrentPolicyFile
128  * </p>
129  * @since 2.0
130  * @see ProtectionDomain
131  * @see Policy
132  * @see ConcurrentPolicyFile
133  * @see net.jini.security.policy.PolicyFileProvider
134  * @see CachingSecurityManager
135  * @see RemotePolicy
136  */

172     /**
173      * Creates a new <code>DynamicPolicyProvider</code> instance that wraps a
174      * default underlying policy.  The underlying policy is created as follows:
175      * if the
176      * <code>net.jini.security.policy.DynamicPolicyProvider.basePolicyClass</code>
177      * security property is set, then its value is interpreted as the class
178      * name of the base (underlying) policy provider; otherwise, a default
179      * class name of
180      * <code>"net.jini.security.policy.PolicyFileProvider"</code>
181      * is used.  The base policy is then instantiated using the no-arg public
182      * constructor of the named class.  If the base policy class is not found,
183      * is not instantiable via a public no-arg constructor, or if invocation of
184      * its constructor fails, then a <code>PolicyInitializationException</code>
185      * is thrown.
186      * <p>
187      * Note that this constructor requires the appropriate
188      * <code>"getProperty"</code> {@link java.security.SecurityPermission} to
189      * read the
190      * <code>net.jini.security.policy.DynamicPolicyProvider.basePolicyClass</code>
191      * security property, and may require <code>"accessClassInPackage.*"</code>
192      * {@link RuntimePermission}s, depending on the package of the base policy
193      * class.
194      *
195      * @throws  PolicyInitializationException if unable to construct the base
196      *          policy
197      * @throws  SecurityException if there is a security manager and the
198      *          calling context does not have adequate permissions to read the
199      *          <code>net.jini.security.policy.DynamicPolicyProvider.basePolicyClass</code>
200      *          security property, or if the calling context does not have
201      *          adequate permissions to access the base policy class
202      */
203     public DynamicPolicyProvider() throws PolicyInitializationException{
204         String cname = Security.getProperty(basePolicyClassProperty);
205 	if (cname == null) {
206 	    cname = defaultBasePolicyClass;
207 	}
208         String tRue = "TRUE";
209         String revoke = Security.getProperty(revocationSupported);
210         if (revoke == null){
211             revoke = tRue;
212         }
213 	try {
214 	    basePolicy = (Policy) Class.forName(cname).newInstance();
215 	} catch (SecurityException e) {
216 	    throw e;
217 	} catch (Exception e) {
218 	    throw new PolicyInitializationException(
219 		"unable to construct base policy", e);
220 	}
221         dynamicPolicyGrants = Collections.newSetFromMap(new ConcurrentHashMap<PermissionGrant,Boolean>(64));
222         loggable = logger.isLoggable(Level.FINEST);
223         if (basePolicy instanceof DynamicPolicy) {
224             if (basePolicy instanceof RevocablePolicy ) {
225                 RevocablePolicy rp = (RevocablePolicy) basePolicy;
226                 revocable = rp.revokeSupported();
227             } else {
228                 revocable = false;
229             }
230         } else {
231             revocable = revoke.equals(tRue);
232         }
233         policyPermissions = basePolicy.getPermissions(policyDomain);
234         policyPermissions.setReadOnly();
235     }

247         if (basePolicy == null) throw new NullPointerException("null basePolicy prohibited");
248         this.basePolicy = basePolicy;
249         dynamicPolicyGrants = Collections.newSetFromMap(new ConcurrentHashMap<PermissionGrant,Boolean>(64));
250         loggable = logger.isLoggable(Level.FINEST);
251          if (basePolicy instanceof DynamicPolicy) {
252             if (basePolicy instanceof RevocablePolicy ) {
253                 RevocablePolicy rp = (RevocablePolicy) basePolicy;
254                 revocable = rp.revokeSupported();
255             } else {
256                 revocable = false;
257             }
258         } else {
259             revocable = true;
260         }
261         policyPermissions = basePolicy.getPermissions(policyDomain);
262         policyPermissions.setReadOnly();
263     }

326 	/* It is extremely important that dynamic grant's are not returned,

336 //        if (domain == policyDomain) return policyPermissions;
337 	/* Note: we can return revokeable permissions, the  ProtectionDomain
338          * only temporarily merges the permissions for toString(), for debugging.
339 	 */
340         Collection<PermissionGrant> pgc = getPermissionGrants(domain);  
341         NavigableSet<Permission> permissions = new TreeSet<Permission>(comparator);

348     @Override
349     public boolean implies(ProtectionDomain domain, Permission permission) {
350         if (domain == policyDomain) return policyPermissions.implies(permission);
351         // There's not much point to this check anymore, besides, it could allow 
352         // a less privileged domain in an underlying policy to be checked
353         // prior to a privileged domain in this policy.
354 //        if (basePolicyIsDynamic || basePolicyIsRemote){
355 //            if (basePolicy.implies(domain, permission)) return true;
356 //        }
357 	if (permission == null) throw new NullPointerException("permission not allowed to be null");
358         /* If com.sun.security.provider.PolicyFile:
359          * Do not call implies on the base Policy, if
360          * there are UnresolvedPermission's that are undergoing resolution
361          * while another Permission within that collection is already
362          * resolved, the Enumeration will cause a ConcurrentModificationException.
363          */ 
364         
365         /* Be mindful of static Permissions held by the 
366          * ProtectionDomain, a Permission may be implied by the 
367          * the combination of Permission's in the ProtectionDomain and 
368          * the base policy, but not by either individually.
369          * The ProtectionDomain merge is only perfomed if
370          * ProtectionDomain.toString() is called, this is purely for debugging
371          * the policy permissions are never merged back into the
372          * ProtectionDomain, the underlying policy
373          * performs the merge.
374          * 
375          * Furthermore it is commonly understood that when
376          * ProtectionDomain.implies(Permission) is called, it first checks
377          * it's own private Permissions, then calls Policy.implies, however
378          * this is incorrect, the Policy is checked first.
379          */ 
380        /* Don't use the underlying policy permission collection otherwise
381         * we can leak grants in to the underlying policy from our cache,
382         * this could then be inadvertantly cached and passed to a ProtectionDomain
383         * constructor, preventing Revocation.
384         */
385         NavigableSet<Permission> permissions = new TreeSet<Permission>(comparator); // Keep as small as possible.
386         /* If GrantPermission is being requested, we must get all Permission objects
387          * and add them to the underlying collection.
388          * 
389          */
390         Class permClass = permission instanceof GrantPermission ? null : permission.getClass();
391         if (!(basePolicy instanceof ScalableNestedPolicy)) {
392             PermissionCollection pc = basePolicy.getPermissions(domain);
393             Enumeration<Permission> enu = pc.elements();
394             while (enu.hasMoreElements()){
395                 Permission p = enu.nextElement();
396                 if (p instanceof AllPermission) return true; // Return early.
397                 if ( permClass == null){
398                     permissions.add(p);
399                 } else if ( permClass.isInstance(permission) || permission instanceof UnresolvedPermission){
400                     permissions.add(p);
401                 }
402             }
403         }else{
404             List<PermissionGrant> grants = ((ScalableNestedPolicy) basePolicy).getPermissionGrants(domain);
405             if ( !grants.isEmpty() && grants.get(0).isPrivileged()) return true;
406             processGrants(grants, permClass, false, permissions);
407         }
408         Iterator<PermissionGrant> grants = dynamicPolicyGrants.iterator();
409         // Check the privileged case first.
410         while (grants.hasNext()){
411             PermissionGrant g = grants.next();
412             if (g.isPrivileged() && g.implies(domain)) return true;
413         }
414         grants = dynamicPolicyGrants.iterator();
415         while (grants.hasNext()){
416             PermissionGrant g = grants.next();
417             if (!g.isPrivileged() && g.implies(domain)){
418                 Collection<Permission> perms = g.getPermissions();
419                 // But we only want to add relevant Permissions.
420                 Iterator<Permission> it = perms.iterator();
421                 while (it.hasNext()){
422                     Permission p = it.next();
423                     if ( permClass == null){
424                         permissions.add(p);
425                     } else if ( permClass.isInstance(permission) || permission instanceof UnresolvedPermission){
426                         permissions.add(p);
427                     }
428                 }
429             }
430         }   
431         PermissionCollection pc = null;
432         if (permClass != null){
433             pc =convert(permissions, 4, 0.75F, 1, 2);
434         } else {
435             // GrantPermission
436             pc = convert(permissions, 4, 0.75F, 1, 2);
437             expandUmbrella(pc);
438         }
439         return pc.implies(permission);
440     }

442     /**
443      * Calling refresh doesn't remove any dynamic grant's, it only clears
444      * the cache and refreshes the underlying Policy, it also removes any
445      * grants for ProtectionDomains that no longer exist.
446      * 
447      * If a CachingSecurityManager has been set, this method will clear its 
448      * cache.
449      * 
450      */
451     
452     @SuppressWarnings("unchecked")
453     public void refresh() {
454         basePolicy.refresh();
455         // Clean up any void dynamic grants.
456         Collection<PermissionGrant> remove = new LinkedList<PermissionGrant>();
457 	Iterator<PermissionGrant> i = dynamicPolicyGrants.iterator();
458         while (i.hasNext()){
459             PermissionGrant p = i.next();
460             if (p.isVoid()){
461                 remove.add(p);
462             }
463         }
464         dynamicPolicyGrants.removeAll(remove);
465         // Don't bother removing void from the remotePolicy, it get's replaced anyway.
466         // Policy file based grant's don't become void, only dynamic grant's
467         // to ProtectionDomain or ClassLoader.
468         final SecurityManager sm = System.getSecurityManager();
469         if (sm != null){
470             if (sm instanceof CachingSecurityManager){
471                 ((CachingSecurityManager) sm).clearCache();
472             }
473         }
474         
475     }

481     public void grant(final Class cl, Principal[] principals, Permission[] permissions) {
482         if (principals == null){ principals = new Principal[0];}
483         checkNullElements(principals);
484         // This has to be after checkNullElements principals or we fail the NullCases test.
485         if (permissions == null || permissions.length == 0) {return;}
486         checkNullElements(permissions);
487         // Not delgated to base policy.
488         Guard g = new GrantPermission(permissions);
489         g.checkGuard(null);
490         final PermissionGrantBuilder pgb = PermissionGrantBuilder.newBuilder();
491         pgb.principals(principals)
492             .permissions(permissions)
493             .context(PermissionGrantBuilder.CLASSLOADER);
494         if (cl != null){
495             AccessController.doPrivileged(
496                 new PrivilegedAction(){
497 
498                     public Object run() {
499                         pgb.clazz(cl);
500                         return null;
501                     }
502 
503                 });
504         }
505         PermissionGrant pe = pgb.build();
506 	dynamicPolicyGrants.add(pe);
507 	if (loggable){
508 	    logger.log(Level.FINEST, "Granting: {0}", pe.toString());
509 	}
510     }

512     // documentation inherited from DynamicPolicy.getGrants
513     public Permission[] getGrants(Class cl, Principal[] principals) {
514         ClassLoader loader = null;
515         if( cl != null ) {
516             loader = cl.getClassLoader();
517         }
518         // defensive copy array
519         if (principals != null && principals.length > 0) {
520 	    principals = principals.clone();
521 	    checkNullElements(principals);
522 	}
523         Collection<Permission> dPerms = new HashSet<Permission>();
524         Iterator<PermissionGrant> grants = dynamicPolicyGrants.iterator();
525 	while ( grants.hasNext()){
526             PermissionGrant g = grants.next();
527 	    if ( g.implies(loader, principals) ){
528 		// Only use the trusted grantCache.
529 		dPerms.addAll(g.getPermissions());
530 	    }
531 	}	
532         Permission[] perms = dPerms.toArray(new Permission[dPerms.size()]);        
533         return perms;
534     }

536     public List<PermissionGrant> getPermissionGrants(ProtectionDomain domain) {
537         List<PermissionGrant> grants = null;
538         if (basePolicy instanceof ScalableNestedPolicy){
539             grants = ((ScalableNestedPolicy)basePolicy).getPermissionGrants(domain);
540             if ( !grants.isEmpty() && grants.get(0).isPrivileged()) return grants;
541         } else {
542             grants = new LinkedList<PermissionGrant>();
543             PermissionGrant pg = extractGrantFromPolicy(basePolicy, domain);
544             grants.add(pg);
545             if (pg.isPrivileged()) return grants;
546         }
547         Iterator<PermissionGrant> it = dynamicPolicyGrants.iterator();
548         while (it.hasNext()){
549             PermissionGrant pg = it.next();
550             /* Check privileged first */
551             if (pg.isPrivileged() && pg.implies(domain)){
552                 grants.clear();
553                 grants.add(pg);
554                 return grants;
555             }
556         }
557         it = dynamicPolicyGrants.iterator();
558         while (it.hasNext()){
559             PermissionGrant pg = it.next();
560             /* Then check less privileged domains*/
561             if ( !pg.isPrivileged() && pg.implies(domain)){
562                 grants.add(pg);
563             }
564         }
565         return grants;
566     }

568     public boolean grant(PermissionGrant p) {
569         Collection<Permission> perms = p.getPermissions();
570         GrantPermission guard = new GrantPermission(perms.toArray(new Permission [perms.size()]));
571         guard.checkGuard(null);
572         return dynamicPolicyGrants.add(p);
573     }
