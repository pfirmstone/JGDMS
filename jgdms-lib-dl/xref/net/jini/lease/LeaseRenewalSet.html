26  /**
27   * A collection of leases being managed by a lease renewal service.
28   * <p>
29   * Clients of the renewal service organize the leases they wish to
30   * have renewed into <em>lease renewal sets</em> (or <em>sets</em>,
31   * for short). These sets are represented by objects implementing this
32   * interface. The
33   * <code>LeaseRenewalService.createLeaseRenewalSet</code> method is
34   * provided to create sets. Sets are populated by methods on the sets
35   * themselves. Two leases in the same set need not be granted by the
36   * same service or have the same expiration time; in addition, they
37   * can be added or removed from the set independently.
38   * <p>
39   * This interface is not a remote interface; each implementation of the
40   * renewal service exports proxy objects that implement the
41   * <code>LeaseRenewalSet</code> interface that use an
42   * implementation-specific protocol to communicate with the actual
43   * remote server. All of the proxy methods obey normal RMI remote
44   * interface semantics except where explicitly noted. Two proxy objects
45   * are equal if they are proxies for the same set created by the same
46   * renewal service. Every method invocation (on both a
47   * <code>LeaseRenewalService</code> and all the
48   * <code>LeaseRenewalSet</code> instances created by that server) is
49   * atomic with respect to other invocations.
50   * <p>
51   * A number of the methods in this class throw
52   * <code>RemoteException</code>, each of these may throw the
53   * <code>java.rmi.NoSuchObjectException</code> subclass. If a client
54   * receives a <code>NoSuchObjectException</code> when calling a method
55   * on a renewal set, the client can infer that the set has been
56   * destroyed; however, it should not infer that the renewal service
57   * has been destroyed.
58   * <p>
59   * The term <em>client lease</em> is used to refer to a lease that has
60   * been placed into a renewal set. Client leases are distinct from the
61   * leases that the renewal service grants on renewal sets it has
62   * created.
63   * <p>
64   * Each client lease has two expiration related times associated with
65   * it: the <em>desired expiration</em> time for the lease, and the
66   * <em>actual expiration</em> time granted when the lease was created
67   * or last renewed. The desired expiration represents when the client
68   * would like the lease to expire. The actual expiration represents
69   * when the lease is going to expire if it is not renewed. Both time
70   * values are absolute times, not relative time durations. When a
71   * client lease's desired expiration arrives, the lease will be
72   * removed from the set without further client intervention.
73   * <p>
74   * Each client lease also has two other associated attributes: a
75   * desired <em>renewal duration</em> and a <em>remaining desired
76   * duration</em>. The desired renewal duration is specified by the
77   * client (directly or indirectly) when the lease is added to the
78   * set. This duration must normally be a positive number, however, it
79   * may be <code>Lease.ANY</code> if the lease's desired expiration is
80   * <code>Lease.FOREVER</code>. The remaining desired duration is
81   * always the desired expiration less the current time.
82   * <p>
83   * Each time a client lease is renewed, the renewal service will ask
84   * for an extension equal to the lease's renewal duration if the
85   * renewal duration is:
86   * <ul>
87   *     <li> Lease.ANY, or 
88   *     <li> less than the remaining desired duration, 
89   * </ul>
90   * 
91   * otherwise, it will ask for an extension equal to the lease's
92   * remaining desired duration.
93   * <p>
94   * If a client lease's actual expiration is later than its desired
95   * expiration, the renewal service will not renew the lease; the lease
96   * will remain in the set until its desired expiration is reached, the
97   * set is destroyed, or it is removed by the client.
98   * <p>
99   * Each set is leased from the renewal service. If the lease on a set
100  * expires or is cancelled, the renewal service will destroy the set
101  * and take no further action with regard to the client leases in the
102  * set. Each lease renewal set has associated with it an expiration
103  * warning event that occurs at a client-specified time before the
104  * lease on the set expires. Clients can register for warning events
105  * using methods provided by the set. A registration for warning
106  * events does not have its own lease, but instead is covered by the
107  * same lease under which the set was granted.
108  * <p>
109  * The term <em>definite exception</em> is used to refer to an
110  * exception that could be thrown by an operation on a client lease
111  * (such as a remote method call) that would be indicative of a
112  * permanent failure of the client lease. For purposes of this
113  * document, all bad object exceptions, bad invocation exceptions, and
114  * <code>LeaseException</code>s are considered to be definite
115  * exceptions.
116  * <p>
117  * Each lease renewal set has associated with it a renewal failure
118  * event that will occur in either of two cases: if any client lease
119  * in the set reaches its actual expiration before its desired
120  * expiration is reached, or if the renewal service attempts to renew
121  * a client lease and gets a definite exception. Clients can register
122  * for failure events using methods provided by the set. A
123  * registration for failure event does not have its own lease, but
124  * instead is covered by the same lease under which the set was
125  * granted.
126  * <p>
127  * Once placed in a set, a client lease will stay there until one or
128  * more of the following occurs:
129  * <ul>
130  *     <li> The lease on the set itself expires or is cancelled,
131  *          causing destruction of the set
132  *     <li> The client lease is removed by the client
133  *     <li> The client lease's actual or desired expiration is reached
134  *     <li> A renewal attempt on the client lease results in a
135  *          definite exception
136  * </ul>
137  * <p>
138  * Each client lease in a set will be renewed as long as it is in the
139  * set. If a renewal call throws an indefinite exception, the renewal
140  * service should retry the lease renewal until the lease would
141  * otherwise be removed from the set. The renewal service will never
142  * cancel a client lease. The preferred method of cancelling a client
143  * lease is for the client to first remove the lease from the set and
144  * then call cancel on it. It is also permissible for the client to
145  * cancel the lease without first removing the lease from the set,
146  * although this is likely to result in additional network traffic.
147  * <p>
148  * Client leases get returned to clients in a number of ways (via
149  * <code>remove</code> and <code>getLeases</code> calls, as components
150  * of events, etc.). In general, they should have their serial format
151  * set to <code>Lease.DURATION</code> before being transferred to the
152  * client's virtual machine. In some exceptional circumstances, this
153  * may not be possible (for example, the client lease was recovered
154  * from persistent storage and could not be deserialized in the
155  * server's virtual machine). In these cases, it is acceptable to
156  * transfer the lease to the client using the
157  * <code>Lease.ABSOLUTE</code> serial format.
158  * <p>
159  * Whenever a client lease gets returned to a client, its actual
160  * expiration should reflect either:
161  * <ul>
162  *     <li> The result of the last successful renewal call that the
163  *          renewal service made or
164  *     <li> The expiration time the lease originally had when it was
165  *          added to the set, if the renewal service has not yet
166  *          successfully renewed the lease
167  * </ul>
168  * 
169  * @author Sun Microsystems, Inc.
170  * @see LeaseRenewalService
171  */
172 public interface LeaseRenewalSet {
173     /**
174      * The event id for all <code>RenewalFailureEvent</code> objects.
175      *
176      * @see RenewalFailureEvent
177      */
178     final static public long RENEWAL_FAILURE_EVENT_ID = 0;

181      * The event id for all <code>ExpirationWarningEvent</code> objects.

183      * <code>see ExpirationWarningEvent</code>
184      */
185     final static public long EXPIRATION_WARNING_EVENT_ID = 1;

187     /**
188      * Include a client lease in the set for a specified duration and
189      * with a specified renewal duration.
190      * <p>
191      * The <code>leaseToRenew</code> argument specifies the lease to be
192      * added to the set. An <code>IllegalArgumentException</code> must
193      * be thrown if the lease was granted by the renewal service
194      * itself. If <code>leaseToRenew</code> is <code>null</code>, a
195      * <code>NullPointerException</code> must be thrown.
196      * <p>
197      * The <code>desiredDuration</code> argument is the number of
198      * milliseconds the client would like the lease to remain in the
199      * set. It is used to calculate the lease's initial desired
200      * expiration by adding <code>desiredDuration</code> to the current
201      * time (as viewed by the service). If this causes an overflow, a
202      * desired expiration of <code>Long.MAX_VALUE</code> will be
203      * used. Unlike a lease duration, the desired duration is
204      * unilaterally specified by the client, not negotiated between the
205      * client and the service. Note, a negative value for
206      * <code>desiredDuration</code> (including <code>Lease.ANY</code>)
207      * will result in a desired expiration that is in the past, causing
208      * <code>leaseToRenew</code> to be dropped from the set; this action
209      * will neither result in an exception or an event.
210      * <p>
211      * If the actual expiration time of <code>leaseToRenew</code> is
212      * less than the current time (as viewed by the renewal service) and
213      * the current time is less than the desired expiration time for
214      * <code>leaseToRenew</code>, the method will return
215      * normally. However, <code>leaseToRenew</code> will be dropped from
216      * the set and a renewal failure event will be generated.
217      * <p>
218      * The <code>renewDuration</code> is the initial renewal duration to
219      * associate with <code>leaseToRenew</code> (in milliseconds). If
220      * <code>desiredDuration</code> is exactly
221      * <code>Long.MAX_VALUE</code>, the <code>renewDuration</code> may
222      * be any positive number or <code>Lease.ANY</code>; otherwise it
223      * must be a positive number. If these requirements are not met, the
224      * renewal service must throw an
225      * <code>IllegalArgumentException</code>.
226      * <p>
227      * Calling this method with a lease that is equivalent to a client
228      * lease already in the set will associate the existing client lease
229      * in the set with the new desired duration and renew duration. The
230      * client lease is not replaced because it is more likely that the
231      * renewal service, rather than the client, has an up-to-date lease
232      * expiration. The service is more likely to have an up-to-date
233      * expiration because the client should not be renewing a lease that
234      * it has passed to a lease renewal service unless the lease is
235      * removed first. These semantics also allow <code>renewFor</code>
236      * to be used in an idempotent fashion.
237      *
238      * @param leaseToRenew the lease to be added to the renewal set
239      * @param desiredDuration the maximum length of time in milliseconds
240      *	      the <code>leaseToRenew</code> should remain in the set, or
241      *	      <code>Lease.FOREVER</code> which implies there is no
242      *	      pre-specified time when the lease should be removed from
243      *	      the set
244      * @param renewDuration the lease duration to request when renewing
245      *	      the lease, unless <code>renewDuration</code> is greater
246      *	      than the remainder of the <code>desiredDuration</code>
247      * @throws IllegalArgumentException if <code>desiredDuration</code>
248      *	       is <code>Lease.FOREVER</code>, <code>renewDuration</code>
249      *	       is not a positive value, <code>Lease.FOREVER</code>, or
250      *	       <code>Lease.ANY</code>. If <code>desiredDuration</code>
251      *	       is not <code>Lease.FOREVER</code>,
252      *	       <code>IllegalArgumentException</code> will be thrown if
253      *	       <code>renewDuration</code> is not a positive value or
254      *	       <code>Lease.ANY</code>. <code>IllegalArgumentException</code>
255      *	       will also be thrown if <code>leaseToRenew</code> 
256      *	       was granted by this renewal service.
257      * @throws NullPointerException if <code>leaseToRenew</code> is
258      *	       <code>null</code>
259      * @throws RemoteException if a communication-related exception
260      *	       occurs
261      */
262     public void renewFor(Lease leaseToRenew, long desiredDuration,
263 			 long  renewDuration)
264 	throws RemoteException;
265 
266     /**
267      * Include a client lease in the set for a specified duration.
268      * <p>
269      * Calling this method is equivalent to making the following call 
270      * on this set:
271      *
272      * <pre>
273      *     renewFor(leaseToRenew, desiredDuration, Lease.FOREVER)
274      * </pre>
275      *
276      * @param leaseToRenew the lease to be added to the renewal set
277      * @param desiredDuration the maximum length of time in milliseconds
278      *	      the <code>leaseToRenew</code> should remain in the set, or
279      *	      <code>Lease.FOREVER</code> which implies there is no
280      *	      pre-specified time when the lease should be removed from
281      *	      the set
282      * @throws IllegalArgumentException if <code>leaseToRenew</code> 
283      *	       was granted by this renewal service
284      * @throws NullPointerException if <code>leaseToRenew</code> is
285      *	       <code>null</code>
286      * @throws RemoteException if a communication-related exception
287      *	       occurs
288      */
289     public void renewFor(Lease leaseToRenew, long desiredDuration)
290 	throws RemoteException;

292     /**
293      * Removes the specified lease from set. If the lease is currently
294      * in the set it will be returned, otherwise <code>null</code> will
295      * be returned. <code>leaseToRemove</code> will not be 
296      * canceled by this call.
297      *
298      * @param leaseToRemove lease to be removed from the set
299      * @return the removed lease if it was in the set or
300      *	       <code>null</code> if it was not
301      * @throws NullPointerException if <code>leaseToRemove</code> is
302      *	       <code>null</code>
303      * @throws RemoteException if a communication-related exception
304      *	       occurs
305      */
306     public Lease remove(Lease leaseToRemove) 
307 	throws RemoteException;

309     /**
310      * Returns all the leases currently in the set. 
311      *
312      * @return the leases in the set. Return a zero length array if
313      *	       there are not leases currently in the set.
314      * @throws LeaseUnmarshalException if one or more of the leases can
315      *	       not be unmarshalled. The throwing of a
316      *	       <code>LeaseUnmarshalException</code> represents a,
317      *	       possibly transient, failure in the ability to unmarshal
318      *	       one or more client leases in the set, it does not
319      *	       necessarily imply anything about the state of the renewal
320      *	       service or the set themselves.
321      * @throws RemoteException if a communication-related exception
322      *	       occurs
323      */
324     public Lease[] getLeases() throws LeaseUnmarshalException, RemoteException;

326     /**
327      * Register for the expiration warning event associated with this
328      * set.
329      * <p>
330      * This method allows the client to register for notification of
331      * the approaching expiration of the set's lease. The
332      * <code>listener</code> argument specifies what listener should
333      * be notified when the lease is about to expire. The
334      * <code>minWarning</code> argument specifies a minimum number of
335      * milliseconds before lease expiration that the first event
336      * delivery attempt should be made by the service. The service may
337      * also make subsequent delivery attempts if the first and any
338      * subsequent attempts have been indeterminate. The
339      * <code>minWarning</code> argument must be zero or a positive
340      * number; if it is not, an <code>IllegalArgumentException</code>
341      * must be thrown. If the current expiration of the set's lease is
342      * less than <code>minWarning</code> milliseconds away, the event
343      * will occur immediately (though it will take time to propagate
344      * to the listener).
345      * <p>
346      * The <code>handback</code> argument to this method specifies an
347      * object that will be part of the expiration warning event
348      * notification. This mechanism is detailed in the Jini
349      * Distributed Event Specification.
350      * <p>
351      * This method returns the event registration for this event. The
352      * <code>Lease</code> object associated with the registration will
353      * be equivalent (in the sense of <code>equals</code>) to the
354      * <code>Lease</code> on the renewal set. Because the event
355      * registration shares a lease with the set, clients that want to
356      * just remove their expiration warning registration without
357      * destroying the set should use the
358      * <code>clearExpirationWarningListener</code> method instead of
359      * cancelling the registration's lease. The registration's event
360      * ID will be
361      * <code>LeaseRenewalSet.EXPIRATION_WARNING_EVENT_ID</code>. The
362      * source of the registration will be the set. The method must
363      * throw a <code>NullPointerException</code> if the
364      * <code>listener</code> argument is <code>null</code>. If an
365      * event handler has already been specified for this event the
366      * current registration is replaced with the new one. Because both
367      * registrations are for the same kind of event, the events sent
368      * to the new registration must be in the same sequence as the
369      * events sent to the old registration.
370      *
371      * @param listener the listener to be notified when this event
372      *	      occurs
373      * @param minWarning how long before the lease on the set expires
374      *	      should the event be sent
375      * @param handback an object to be handed back to the listener when
376      *	      the warning event occurs
377      * @return an <code>EventRegistration</code> describing the event
378      *	       registration
379      * @throws IllegalArgumentException if <code>minWarning</code> is
380      *	       negative
381      * @throws NullPointerException if <code>listener</code> is
382      *	       <code>null</code>
383      * @throws RemoteException if a communication-related exception
384      *	       occurs
385      */
386     public EventRegistration setExpirationWarningListener(
387 	                         RemoteEventListener listener, 
388 				 long                minWarning, 
389 				 MarshalledObject    handback)
390 	throws RemoteException;
391 
392     /**
393      * Remove the listener currently registered for expiration warning
394      * events. It is safe to call this method even if no listener is
395      * currently registered.
396      *
397      * @throws RemoteException if a communication-related exception
398      *	       occurs
399      */
400     public void clearExpirationWarningListener()
401 	throws RemoteException;

403     /**
404      * Register for the renewal failure event associated with this set.
405      * <p>    
406      * This method allows the client to register for the event
407      * associated with the failure to renew a client lease in the
408      * set. These events are generated when a client lease expires
409      * while it is still in the set, or when the service attempts to
410      * renew a client lease and gets a definite exception. The
411      * listener argument specifies the listener to be notified if a
412      * client lease could not be renewed.
413      * <p>
414      * The <code>handback</code> argument specifies an object that
415      * will be part of the renewal failure event notification. This
416      * mechanism is detailed in the Jini Distributed Event
417      * Specification.
418      * <p>
419      * This method returns the event registration for this event. The
420      * <code>Lease</code> object associated with the registration will
421      * be equivalent (in the sense of <code>equals</code>) to the
422      * <code>Lease</code> on the renewal set. Because the event
423      * registration shares a lease with the set, clients that want to
424      * just remove their expiration warning registration without
425      * destroying the set should use the
426      * <code>clearRenewalFailureListener</code> method, instead of
427      * cancelling the registration's lease. The registration's event
428      * ID will be
429      * <code>LeaseRenewalSet.RENEWAL_FAILURE_EVENT_ID</code>. The
430      * source of the registration will be the set. The method must
431      * throw <code>NullPointerException</code> if the
432      * <code>listener</code> argument is <code>null</code>. If an
433      * event handler has already been specified for this event the
434      * current registration is replaced with the new one. The returned
435      * event registration must have the same event ID as the replaced
436      * registration. Because both registrations are for the same kind
437      * of event, the events sent to the new registration must be in
438      * the same sequence as the events sent to the old registration.
439      *
440      * @param listener the listener to be notified when this event
441      *	      occurs
442      * @param handback an object to be handed back to the listener when
443      *	      the warning event occurs
444      * @return an <code>EventRegistration</code> describing the event
445      *	      registration
446      * @throws NullPointerException if <code>listener</code> is
447      *	       <code>null</code>
448      * @throws RemoteException if a communication-related exception
449      *	       occurs
450      */
451     public EventRegistration setRenewalFailureListener(
452 				 RemoteEventListener listener, 
453 				 MarshalledObject    handback)
454 	throws RemoteException;
455 
456     /**
457      * Remove the listener currently registered for renewal failure
458      * events. It is safe to call this method even if no listener is
459      * currently registered.
460      *
461      * @throws RemoteException if a communication-related exception
462      *	       occurs
463      */
464     public void clearRenewalFailureListener()
465 	throws RemoteException;
466 
467     /**
468      * Returns the lease that controls the lifetime of this set. Can
469      * be used to extend or end the sets lifetime. Note that this method
470      * does not make a remote call.
471      *
472      * @return the lease that controls the lifetime of this set
473      */
474     public Lease getRenewalSetLease();
